# Performance Optimization

1. Memoization:
   Use React.memo, useMemo, and useCallback to prevent unnecessary renders.

2. Code Splitting:
   Implement route-based and component-based code splitting using tools like Webpack or Vite.

3. Avoid Prop Drilling:
   Use Context API or global state management to pass data across deeply nested components.

4. Debouncing and Throttling:
   Use lodash or custom hooks to optimize high-frequency events like search or scroll.

5. Batching Updates:
   Leverage React’s automatic batching in concurrent mode.

6. Using Virtualization and pagination and infinite scroll for large lists.

7. **Debouncing** = “Wait Until Knocking Stops”
   Use of debouncing for search inputs which requires network calls
   Door knocking analogy for debouncing
   Someone keeps knocking the door, You don’t open the door immediately. Only after the knocking stops for 5 seconds, you open the door. If they keep knocking, the timer resets.

   Debounce Meaning: (Execute action only after activity stops for a certain time.)
   In search input:
   1. User types: r, re, rea, reac, react
   2. You wait 500ms after typing stops
   3. Then call API once
      No matter how many keystrokes → only 1 call.

      Reset timer on every event
      Execute only after delay without interruption

8. 🔥 **Throttling** = “Open Door Every 5 Seconds Max”
   Door knocking analogy for debouncing
   Someone keeps knocking continuously.
   You decide: I will only open the door once every 5 seconds — no matter how much they knock.
   If they knock 50 times in 5 seconds:
   You still open door only once during that interval.
   After 5 seconds → next opening allowed.
   **Throttling Meaning**: Limit execution to once per fixed time interval.
   1. Scroll event firing 100 times per second
   2. You allow function to run only once every 200ms
      Even if user keeps scrolling → limited execution rate.

   | Feature        | Debounce             | Throttle              |
   | -------------- | -------------------- | --------------------- |
   | Trigger timing | After activity stops | At fixed intervals    |
   | Best for       | Search input         | Scroll/resize         |
   | API calls      | Only last one        | Multiple, but limited |
   | Behavior       | Wait → then act      | Act periodically      |

   **Difference between debouncing and throttling?**

   Debouncing delays execution until the event stops firing for a specified time, while throttling limits execution to at most once per defined interval regardless of how often the event fires.

   🔥 When to Use Each
   Use Debounce:
   1. Search input
   2. Auto-save
   3. Form validation
   4. API calls while typing

   Use Throttle:
   1. Scroll tracking
   2. Window resize
   3. Mouse movement
   4. Infinite scroll detection
