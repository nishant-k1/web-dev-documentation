# Build Time FAQs

## So, in react apps if we didn't implement route level code splitting and lazy loading then enter app all pages were loading while we visited homepage?

👉 Yes — if you don’t implement code splitting, the entire dependency graph gets bundled and downloaded on the first page load.

👉 Yes, the entire reachable app code is bundled together and downloaded when the user first loads the application, even though only one route is rendered.

## 🟢 Without Code Splitting

Suppose your app has:

```sql
Home
Dashboard
Settings
AdminPanel
Reports
```

And in App.tsx you do normal imports:

```jsx
import Home from "./Home";
import Dashboard from "./Dashboard";
import Settings from "./Settings";
import AdminPanel from "./AdminPanel";
import Reports from "./Reports";
```

Even if your router shows only <Home /> at

During build:

The bundler sees all static imports.

So it creates:

```sql
main.js (contains Home + Dashboard + Settings + AdminPanel + Reports)
```

Now when user visits /:

Browser downloads:

```sql
HTML
main.js (which includes all pages)
CSS
```

So yes — all page code is downloaded, even if not rendered yet.

## 🟢 Important Clarification

Downloaded ≠ Rendered

Even though all pages are in the bundle:

- Only Home component renders
- Other components just sit in memory

But they still:

- Increase bundle size
- Increase download time
- Increase parse & execution time

## 🟢 With Route-Level Code Splitting

If you change to:

```jsx
const Dashboard = React.lazy(() => import("./Dashboard"));
```

Now bundler creates:

```jsx
main.js (Home + router logic)
dashboard.chunk.js
settings.chunk.js
admin.chunk.js
```

When user visits /:

- Only main.js loads.
- Dashboard code is fetched only when user navigates there.
- This reduces initial load time significantly.

## Downloaded what do you mean by that? Does it mean like for css external stylesheet browser makes request when that style element runs in the html similarly when script element runs the request is made for js file or js module which when gets downloaded?

Yes — you’re thinking correctly. Let’s slow it down and visualize what “download” actually means in a React app.

When I say download, I mean:

The browser makes a network request to the server to fetch the JavaScript file before it can execute it.

Exactly like CSS.

## 🟢 How Browser Loads a Page (Step by Step)

When user visits: `https://yourapp.com`

1️⃣ Server returns HTML

```html
<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="/assets/style.css" />
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/assets/main.js"></script>
  </body>
</html>
```

2️⃣ Browser Parses HTML
While parsing:

- When it sees <link rel="stylesheet">
  → It makes a network request for style.css

- When it sees <script src="main.js">
  → It makes a network request for main.js

This network request is what we mean by download.

🟢 What Happens in React App Without Code Splitting?
Suppose after build:
`main.js = 2MB`

That file contains:

```sql
Home page code
Dashboard code
Settings code
Admin code
Everything
```

When the browser sees:

```html
<script src="main.js"></script>
```

It makes one network request and downloads the full 2MB.
Even if user is on homepage only.
That’s the download.

🟢 With Code Splitting

Now suppose build output is:

```sql
main.js = 500KB
dashboard.chunk.js = 300KB
settings.chunk.js = 250KB
```

When user visits /:

Browser downloads only:

```sql
main.js (500KB)
```

Later, when user clicks Dashboard:
React dynamically triggers: `import("./Dashboard")`

Browser then makes another network request:

→ downloads dashboard.chunk.js

That’s lazy loading.

## 🟢 Important: Script Execution vs Download

Download happens when browser requests the file.

This is purely a **network activity**.

- Browser sends HTTP request
- Server responds with the file
- File gets downloaded

No execution happens yet.

Execution happens **after download**:

1. File is downloaded
2. JS engine parses it
3. Engine compiles it internally (JIT)
4. Code executes

So:

- Download = Network work
- Parse & Execute = CPU work

Both affect performance, but in different ways.

## 🟢 CSS vs JS Similarity

Your comparison is correct.

```html
<link rel="stylesheet" href="styles.css" />
```

→ Downloads CSS

```html
<script src="app.js"></script>
```

→ Downloads JS

```js
import("./dashboard");
```

→ Downloads JS chunk dynamically

All three use the same mechanism:

✅ HTTP request  
✅ File download

The only difference is **when** and **how** they execute.
