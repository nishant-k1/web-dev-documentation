# When to Use `useEffect` vs Normal Logic in React

## Core Principle

Use **`useEffect` only when you need to synchronize your component with something outside React.**

The important phrase is **"outside React"**.

Examples of outside systems:

- Network / APIs
- Browser APIs
- DOM events
- Timers
- Local storage
- Subscriptions

If your logic **does not interact with an external system**, you usually **do not need `useEffect`**.

---

# 1. Cases Where `useEffect` IS Required

These involve **side effects**.

## API Calls

```javascript
useEffect(() => {
  fetchUsers();
}, []);
```

Reason: Network request is an external operation.

---

## Event Listeners

```javascript
useEffect(() => {
  const handleResize = () => console.log(window.innerWidth);

  window.addEventListener("resize", handleResize);

  return () => window.removeEventListener("resize", handleResize);
}, []);
```

Reason: Browser event system is external to React.

---

## Timers

```javascript
useEffect(() => {
  const id = setInterval(() => {
    console.log("tick");
  }, 1000);

  return () => clearInterval(id);
}, []);
```

Reason: Timer system runs outside React.

---

## Local Storage

```javascript
useEffect(() => {
  localStorage.setItem("query", inputQuery);
}, [inputQuery]);
```

Reason: Browser storage is external.

---

# 2. Cases Where `useEffect` Is NOT Needed

If you're just **computing values from existing state or props**, write the logic **directly inside the component**.

These are **pure calculations**.

---

## Filtering Data

```javascript
const filteredData = users.filter((user) => user.name.includes(query));
```

No side effects — only computing a value.

---

## Sorting Data

```javascript
const sortedUsers = [...users].sort((a, b) => a.name.localeCompare(b.name));
```

Pure calculation.

---

## Formatting Data

```javascript
const fullName = `${firstName} ${lastName}`;
```

Just transforming values.

---

## Mapping Lists

```javascript
const listItems = users.map((user) => <User key={user.id} user={user} />);
```

Rendering logic.

---

# 3. Derived State (Common Mistake)

Developers sometimes store computed data in state.

Example:

```javascript
const [filteredData, setFilteredData] = useState([]);

useEffect(() => {
  setFilteredData(users.filter((user) => user.name.includes(query)));
}, [users, query]);
```

This creates problems:

- extra state
- extra re-renders
- synchronization bugs
- unnecessary complexity

---

## Better Approach

Compute the value during render.

```javascript
const filteredData = users.filter((user) => user.name.includes(query));
```

No extra state required.

---

# 4. Mental Rule to Decide

Ask yourself this question:

> If React ran this logic **100 times**, would anything outside React change?

---

## Example: Filtering

```javascript
const filteredData = users.filter(...);
```

Running this 100 times:

- no network calls
- no storage writes
- no external changes

Therefore:

```
No useEffect needed
```

---

## Example: Fetching Data

```javascript
useEffect(() => {
  fetch("/api/users");
}, []);
```

Running this 100 times would trigger **100 network calls**.

Therefore:

```
This belongs in useEffect
```

---

# 5. React Rendering Model

A React component is essentially a function:

```
UI = f(state, props)
```

Whenever state changes:

```
state changes
↓
component re-runs
↓
new UI is calculated
```

Pure calculations (filtering, mapping, sorting) are **expected to run during render**.

---

# 6. When to Use `useMemo`

If a calculation becomes **expensive**, you may optimize it.

Example:

```javascript
const filteredData = useMemo(() => {
  return users.filter((user) => user.name.includes(query));
}, [users, query]);
```

Use this when:

- datasets are large
- calculations are expensive

Note:

```
useMemo is an optimization, not a requirement
```

---

# 7. The Key React Guideline

A common rule used by experienced React developers:

```
If you can compute something during render,
do not store it in state.
```

---

# Final Summary

```
Side effects → useEffect

Pure calculations → normal logic inside component
```

Examples:

```
API calls        → useEffect
Event listeners  → useEffect
Timers           → useEffect
Local storage    → useEffect

Filtering        → normal code
Sorting          → normal code
Mapping          → normal code
Formatting       → normal code
```

---

```
useEffect is for synchronizing React with the outside world.
```
