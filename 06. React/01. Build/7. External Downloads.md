# When user opens a React app, browser may download:

- HTML
- JS bundles
- CSS
- Fonts
- Images
- API data
- Source maps (dev)
- Workers
- WASM
- Third-party scripts
- Manifest
- Service worker

All of these are generally called:

Web resources
or
Network resources

In browser terminology, they are also called:

HTTP resources
Static assets (some of them)
Subresources

## You can group them as:

- Static assets (JS, CSS, images, fonts)
- Document (HTML)
- Data requests (APIs)
- Worker resources
- Third-party resources
- That’s structured thinking.

## What impacts initial load time apart from JS bundle?

Fonts, CSS, third-party scripts, images, and API calls significantly impact load performance, not just JavaScript bundle size.

## Among all these resources, which one blocks rendering the most by default — CSS or JavaScript?

CSS is render-blocking because the browser must build the CSSOM before painting. JavaScript is parser-blocking by default, as it pauses HTML parsing during execution.

✅ CSS is render-blocking
✅ JavaScript is parser-blocking

🟢 1️⃣ CSS (Render-Blocking)

When the browser sees:`<link rel="stylesheet" href="styles.css" />`

It must:

- Download CSS
- Parse CSS
- Build CSSOM

Only after CSSOM + DOM are ready can it build:

👉 Render Tree
👉 Layout
👉 Paint

So CSS blocks rendering of the page.

The browser does this because:
It cannot safely render without knowing styles.
So CSS is render-blocking by default.

🟢 2️⃣ JavaScript (Parser-Blocking)

By default:

1. HTML parsing stops
2. JS is downloaded
3. JS is executed
4. Then HTML parsing resumes

So JS blocks:

👉 HTML parsing

## 🟢 Important Modern Detail: `defer` or `type="module"` attribute

If you use: `<script defer src="main.js"></script>`

Now:

- HTML parsing continues
- JS executes after DOM is ready
- Much better performance

`Modern React builds use type="module" which behaves like defer.`

So it is no longer parser-blocking.

Modern React builds usually use type="module" or defer, so blocking is reduced.

🟢 Which Blocks Rendering More?

In practical terms:
👉 CSS blocks rendering more directly
👉 JS blocks parsing and can indirectly delay rendering

If CSS is slow: Page stays blank
If JS is slow:

1. DOM parsing pauses
2. But some content may still render (depending on structure)

**CSS downloads in parallel and does not block HTML parsing, but it blocks rendering. JavaScript without defer blocks HTML parsing until it finishes downloading and executing.**
