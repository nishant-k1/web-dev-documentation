# Controlled component and form controls

## Controlled component

A react component where React controls the value via state.

The component that renders the form control with a value / checked prop is called controlled - even if the state comes from its parent that is props

Rendering a form control does NOT automatically make a component a controlled component.

A component is controlled if:

The value (or checked/selected state) of a form control is driven by React state (either its own state or via props from a parent).

A component is controlled if at least one form control inside it has its value controlled by React state:

1. value -> for text inputs, textarea, select
2. checked -> for checkbox, radio

And that state can come from
its own useState
or parent state via props

## Controlled component vs uncontrolled component

React doesn't add controls, but it changes how you use them

1. **Controlled component**: A react component where React controls the value via state

   ```jsx
   <input value={state} onChange={(e) => setState(e.target.value)} />
   ```

2. Uncontrolled component: Form control managed by the DOM (ref, no state)

   ```jsx
   <input ref={inputRef} />
   ```

## Form Control which cannot be controlled, file input type

<input type="file"> cannot be a controlled component.
Even in a fully controlled React form, file inputs must remain uncontrolled, and you always rely on onChange + files (and sometimes ref).

This is not a React design choice — it is a browser-enforced security rule.

**Uncontrolled does NOT mean:**

React is unaware
State is not used
You lose form consistency

It ONLY means:
React does not own the input's value
You still store the file in state:

```jsx
const handleChange = (e) => {
  if (e.target.type === "file") {
    setFormData((prev) => ({
      ...prev,
      file: e.target.files[0],
    }));
  }
};
```

**Since you cannot do this:**

```jsx
setFormData({ file: null }); // ❌ visually does nothing
```

**You must use a ref:**

```jsx
const fileRef = useRef(null);

const clearFile = () => {
  fileRef.current.value = "";
  setFormData((prev) => ({ ...prev, file: null }));
};

<input type="file" ref={fileRef} onChange={handleChange} />;
```

✅ All standard form controls can be reset via component state  
❌ The only exception is <input type="file">

```js
<input type="file" name="file" ref={fileRef} onChange={handleChange} />
```
