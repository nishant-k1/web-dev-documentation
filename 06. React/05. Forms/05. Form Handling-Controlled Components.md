# Form Handling in React

In React, all form handling revolves around:

1. Events (onChange, onSubmit)
2. Event object (e)
3. Target (e.target)
4. State (controlled components)

If you master what e.target gives for each control, you're done

## HTMLCollection

HTMLCollection is just the browser's way of returning a group of DOM elements — it's not a JS array, it's its own special type.

**DOM world**: HTMLCollection
**JS world**: Array

HTMLCollection !== Array

The browser returns collection types in many places, not just `selectedOptions`:

```text
document.querySelectorAll('div')   // NodeList  — also needs spread for .map()
element.children                   // HTMLCollection
element.classList                  // DOMTokenList
form.elements                      // HTMLFormControlsCollection
```

## HTML element type → known properties

| HTML Element Type   | Tag        | What You See in Console | Useful Properties                        |
| ------------------- | ---------- | ----------------------- | ---------------------------------------- |
| `HTMLOptionElement` | `<option>` | Option element object   | `.value`, `.text`, `.selected`           |
| `HTMLInputElement`  | `<input>`  | Input element object    | `.value`, `.checked`, `.name`, `.type`   |
| `HTMLSelectElement` | `<select>` | Select element object   | `.value`, `.selectedOptions`, `.options` |

## For your form handler specifically, you only ever need

| Element          | Properties You’ll Use        |
| ---------------- | ---------------------------- |
| Any input        | `.value`, `.name`, `.type`   |
| Checkbox / Radio | `.checked`                   |
| Option           | `.value`                     |
| Select           | `.selectedOptions`, `.value` |
| File Input       | `.files`                     |

## These aren’t new elements—just extra properties that sometimes matter

| Case                             | Useful Property                    | Why it matters                            |
| -------------------------------- | ---------------------------------- | ----------------------------------------- |
| Multi-select `<select multiple>` | `.selectedOptions`                 | `.value` only gives first selected        |
| `<textarea>`                     | `.value`                           | Same as input, but often asked separately |
| Form reset/defaults              | `.defaultValue`, `.defaultChecked` | Useful in edge cases                      |
| Disabled fields                  | `.disabled`                        | Ignored in form submission                |

## Instead of memorizing elements, think like this

**Text-like inputs** → .value
**Boolean inputs (checkbox/radio)** → .checked
**Grouped inputs (select)** → .value or .selectedOptions
**File inputs** → .files

If you remember this pattern, you won’t need to memorize tables.

## e.target.dataset

dataset = a JS object for reading/writing any `data-*` attributes on an element.
So if an element has `data-*`, it gets a .`dataset`.

dataset is useful when:

1. You want to attach extra metadata
2. That is not part of the actual form value
3. You can use `data-*` attributes (and thus element.dataset) on any HTML element, including all form controls.

```js
<input type="text" data-field="email" data-required="true" data-role="admin">
<select data-type="country">
```

```js
const { field, required, type, role } = e.target.dataset;
```

## Events + Target Mapping for form controls

What to read from target

1. **all input types (except checkbox and file type), textarea and single option select (dropdown) - only single option select**
   - event: `onChange`
   - what to read from target: `e.target.value`
2. **checkbox input single option select**
   - event: `onChange`
   - what to read from target: `e.target.checked`

3. **checkbox input multiple option select**
   - event: `onChange`
   - what to read from target: `manual array build`

4. **file input**
   - event: `onChange`
   - what to read from target: `e.target.files`

5. **multiple options select (dropdown)**
   - event: `onChange`
   - what to read from target: `Array.from(e.target.selectedOptions, o => o.value)`

## Events + Target mapping for non form controls

1. button and submit type button

- event: `onClick`
- what to read from target: `Nothing to read but if button has type submit then must do e.preventDefault() to prevent browsers default behaviour`
  Usually nothing to read, but you may read e.target, e.currentTarget, or dataset

1. form - onSubmit

- event: `onSubmit`
- what to read from target: `Nothing to read but do preventDefault() to prevent browsers default behaviour`

bubbling events are default and capture events are rarely used.

## One Handler for all inputs (Interview Gold)

Prefer using one event handler function for all the form controls

```jsx
const [formData, setFormData] = useState({
  name: "",
  email: "",
  password: "",
  agree: false,
  skills: [],
  gender: "",
  file: null,
  hobbies: [],
});
```

```jsx
<input
  type="text"
  name="name"
  value={formData.name}
  onChange={handleChange}
/>

<input
  type="email"
  name="email"
  value={formData.email}
  onChange={handleChange}
/>

<input
  type="password"
  name="password"
  value={formData.password}
  onChange={handleChange}
/>

<input
  type="checkbox"
  name="agree"
  checked={formData.agree}
  onChange={handleChange}
/>

<label>
  <input
    type="checkbox"
    name="skills"
    value="js"
    checked={formData.skills.includes("js")}
    onChange={handleChange}
  />
  JavaScript
</label>

<label>
  <input
    type="checkbox"
    name="skills"
    value="react"
    checked={formData.skills.includes("react")}
    onChange={handleChange}
  />
  React
</label>

<input
  type="radio"
  name="gender"
  value="male"
  checked={formData.gender === "male"}
  onChange={handleChange}
/>

<input
  type="radio"
  name="gender"
  value="female"
  checked={formData.gender === "female"}
  onChange={handleChange}
/>

<input
  type="file"
  name="file"
  onChange={handleChange}
/>

<select
  multiple
  name="hobbies"
  value={formData.hobbies}
  onChange={handleChange}
>
  <option value="1">Option 1</option>
  <option value="2">Option 2</option>
  <option value="3">Option 3</option>
</select>
```

```js
const handleChange = (e) => {
  const { type, name, value, checked, files, multiple, selectedOptions } =
    e.target;

  setFormData((prev) => {
    let finalValue;

    if (type === "checkbox") {
      // Handle checkbox groups (arrays)
      if (Array.isArray(prev[name])) {
        finalValue = checked
          ? [...prev[name], value]
          : prev[name].filter((v) => v !== value);
      } else {
        finalValue = checked;
      }
    } else if (type === "select-multiple") {
      finalValue = Array.from(selectedOptions, (o) => o.value);
    } else if (type === "file") {
      finalValue = multiple ? files : files[0]; // ← your file has this, notes don't
    } else {
      finalValue = value;
    }
    return {
      ...prev,
      [name]: finalValue,
    };
  });
};
```

## Common Controlled Component Form handling Mistakes (VERY IMPORTANT)

1. Not setting value prop for the form control in controlled component
2. Using value for checkbox input type
3. Using value for file input type
4. Using value for select-multiple and using selectedOptions for select-one
5. Forgetting preventDefault which causes page reload
