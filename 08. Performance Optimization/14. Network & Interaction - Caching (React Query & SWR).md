# 🔵 How to Answer: “How do you implement caching in React?”

Caching can be implemented at multiple layers — browser storage, in-memory state, HTTP layer, or using libraries like React Query.

In my projects, I’ve implemented manual caching using localStorage and in-memory techniques to reduce redundant API calls and improve performance. I also understand more advanced caching strategies at the HTTP and library level.

---

# 🟢 1️⃣ Browser-Level Caching (localStorage / sessionStorage)

After fetching API data:

- Store response in `localStorage`
- Save a timestamp
- On next load, check if cached data exists
- Validate expiry
- Return cached data if still valid
- Otherwise refetch and update cache

### Example:

```ts
const CACHE_KEY = "users";
const CACHE_EXPIRY = 5 * 60 * 1000; // 5 minutes

const getUsers = async () => {
  const cached = localStorage.getItem(CACHE_KEY);

  if (cached) {
    const { data, timestamp } = JSON.parse(cached);

    if (Date.now() - timestamp < CACHE_EXPIRY) {
      return data;
    }
  }

  const response = await fetch("/api/users");
  const data = await response.json();

  localStorage.setItem(
    CACHE_KEY,
    JSON.stringify({ data, timestamp: Date.now() }),
  );

  return data;
};
```

### Key Points:

- Reduces unnecessary API calls
- Improves perceived performance
- Requires manual expiry management
- Not suitable for sensitive data

# 🟢 2️⃣ In-Memory Caching

Caching can also be implemented using module-level variables or `useRef`.

This avoids repeated API calls during a single session.

### Example:

```ts
let cache: Record<string, any> = {};

async function fetchData(url: string) {
  if (cache[url]) return cache[url];

  const res = await fetch(url);
  const data = await res.json();

  cache[url] = data;
  return data;
}
```

### Key Points:

- Faster than localStorage
- Cleared on page reload
- Useful for deduplicating requests
- Works well within SPA lifecycle

---

# 🟢 3️⃣ HTTP-Level Caching

Proper caching is often best handled at the HTTP protocol level.

Important mechanisms:

- `Cache-Control`
- `ETag`
- `If-None-Match`
- `304 Not Modified`

This allows the server to determine freshness and prevents unnecessary data transfer.

This is more scalable and cleaner than purely client-side caching.

# 🟢 4️⃣ Library-Based Caching (React Query)

Even though I haven’t used React Query in production yet, I understand that it provides:

- Automatic caching
- Stale-time configuration
- Background refetching
- Request deduplication
- Cache invalidation
- Refetch on window focus

This abstracts most manual caching logic and provides better scalability for large applications.
