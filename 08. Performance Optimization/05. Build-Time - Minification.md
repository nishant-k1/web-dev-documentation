# Build-Time – Minification

Minification reduces JavaScript/CSS size by removing **whitespace, comments, and shortening names** without changing behavior.

---

## Why it matters

- Smaller files → faster download
- Less parse/compile time
- Helps improve LCP and TTI

---

## Common Tools

- Terser (used by Webpack, Vite)
- esbuild / SWC minifiers

---

## Interview Line

> “My production builds always run minification (Terser/esbuild), so the shipped JavaScript and CSS are as small and fast to parse as possible.”
