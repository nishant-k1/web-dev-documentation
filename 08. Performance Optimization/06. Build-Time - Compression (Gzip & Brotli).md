# Build-Time – Compression (Gzip & Brotli)

After bundling and minification, assets are usually **compressed over the wire** using Gzip or Brotli.

---

## Why Compression

- Reduces transfer size dramatically
- Big impact on slow/mobile networks
- Directly improves **LCP** and **TTFB**

---

## Typical Setup

- Configure server / CDN:
  - Enable **Gzip** and **Brotli**
  - Serve precompressed `.gz` / `.br` files where possible
- In frameworks like Next.js, Vite, CRA, this is often handled by:
  - Production server / reverse proxy (NGINX, Cloudflare, etc.)

---

## Interview Line

> “In production, I ensure static assets are served with Gzip/Brotli compression, which significantly reduces bundle transfer size and improves Web Vitals.”
