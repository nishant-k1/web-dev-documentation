# Network & Interaction Optimization – Event Optimization

This corresponds to **“3️⃣ Network & Interaction Optimization”** in the main `index.md`.

Focus: Improve perceived performance and reduce unnecessary API calls and event work.

---

## Debouncing vs Throttling

See detailed notes in:

- `6. Network & Interaction - Event Optimization/1. Debouncing and Throttling.md`

High level:

- **Debouncing** → Wait until the user stops triggering the event for a certain time, then run the handler (good for search input).
- **Throttling** → Run the handler at most once per interval while events keep firing (good for scroll / resize).

---

## Practical Use Cases

- Debounced search input (avoid firing API on every keystroke)
- Throttled scroll handlers (infinite scroll, scroll-based animations)

You can say in interviews:

> “For high-frequency events like scroll and keypress, I use debouncing and throttling to avoid unnecessary network calls and re-renders, which improves perceived performance.”
