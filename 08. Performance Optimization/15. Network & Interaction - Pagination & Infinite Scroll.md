# Network & Interaction – Pagination & Infinite Scroll

Instead of fetching all data at once, **paginate or use infinite scroll**.

---

## Why

- Smaller responses
- Faster initial load
- Better memory usage

---

## Patterns

- Classic numbered pagination
- “Load more” button
- Infinite scroll with:
  - Intersection Observer
  - Throttled scroll handlers

---

## Interview Line

> “To keep network payloads small, I use pagination or infinite scroll so we only fetch the data the user is about to see, not the entire dataset at once.”
