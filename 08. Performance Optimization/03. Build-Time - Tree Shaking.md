# Tree Shaking

## What it is

Tree shaking removes unused exports from the final production bundle.

It works only with:

- ES Modules (`import` / `export`)
- Production builds

---

## Example

```js
// utils.js
export const add = () => {};
export const subtract = () => {};
```

If you import only:

```js
import { add } from "./utils";
```

`subtract` will be removed from the final bundle.

---

## Important Interview Points

- Works only in **production mode**
- Requires **ES Modules**
- Doesn’t work properly with **CommonJS (`require`)**
- `sideEffects` in `package.json` can affect tree shaking
- Libraries with side effects may not be fully tree-shaken

---

## Smart Thing to Say in Interview

> “I prefer named imports instead of importing entire libraries to help bundlers tree-shake effectively.”

## Example

```js
// ❌ Bad
import _ from "lodash";

// ✅ Good
import debounce from "lodash/debounce";
```

This shows you understand how bundlers optimize production builds and how import patterns affect bundle size.
