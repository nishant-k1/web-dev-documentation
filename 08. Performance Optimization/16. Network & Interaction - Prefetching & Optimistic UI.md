# Network & Interaction – Prefetching & Optimistic UI

These techniques improve **perceived performance**.

---

## Prefetching

- Fetch data **before** the user explicitly asks for it.
- Examples:
  - Hovering a link → prefetch route/data
  - Predict next screen and preload resources

Example:

- React lazy loading and code splitting at route level with prefetching / prefetching the route page component’s

1. **✅ Method 1 — Preload Route Bundle Manually**

   Step 1 — Extract the import function

   ```jsx
   // dashboardLoader.ts
   export const loadDashboard = () => import("./Dashboard");
   export const Dashboard = React.lazy(loadDashboard);
   ```

   Step 2 — Use it in route

   ```jsx
   <Route path="/dashboard" element={<Dashboard />} />
   ```

   Step 3 — Prefetch on Hover

   ```jsx
   import { loadDashboard } from "./loadDashboard";
   import { useNavigate } from "react-router-dom";

   function DashboardLink() {
     const navigate = useNavigate();

     return (
       <button
         onMouseEnter={loadDashboard}
         onClick={() => navigate("/dashboard")}
       >
         Go to Dashboard
       </button>
     );
   }
   ```

2. **✅ Method 2 — Using webpackPrefetch (If Using Webpack)**
   If your bundler supports it:
   This tells Webpack:

   “Download this bundle in idle time.”
   The browser fetches it with low priority.
   ⚠️ Works only if your build tool supports it.

   ```jsx
   const Dashboard = React.lazy(
     () => import(/* webpackPrefetch: true */ "./Dashboard"),
   );
   ```

3. **✅ Method 3 — Prefetch on Visibility (Smarter UX)**
   Instead of hover, use Intersection Observer:
   Now when the link scrolls into view → prefetch happens.
   This is more production-grade than hover.

   ```jsx
   useEffect(() => {
     const observer = new IntersectionObserver((entries) => {
       if (entries[0].isIntersecting) {
         dashboardImport();
       }
     });

     observer.observe(document.getElementById("dashboard-link")!);

     return () => observer.disconnect();
   }, []);
   ```

## FAQ

Q: How do you prevent double loading?

Answer:

Dynamic imports are cached by the module system. Calling the import again doesn’t re-download the bundle

## Optimistic UI / Optimistic Updates

- Immediately update the UI **as if** a mutation succeeded.
- Roll back if the server returns an error.
- Great for:
  - Likes
  - Toggles
  - Simple form updates
