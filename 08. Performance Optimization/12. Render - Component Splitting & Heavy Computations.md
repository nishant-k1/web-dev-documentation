# Render – Component Splitting & Heavy Computations

Large, monolithic components and heavy calculations in render can **block the UI**.

---

## Splitting Large Components

- Break big components into smaller ones:
  - Improves readability
  - Allows React to re-render only the parts that changed
  - Enables code splitting for rarely used sections

---

## Avoid Heavy Work in Render

- Don’t run expensive loops or calculations directly in JSX.
- Move heavy work:
  - Into `useMemo`
  - Into event handlers
  - Off the main thread via **Web Workers** when needed

---

## Interview Line

> “I split large components into smaller units and move heavy computations out of the render path (or into Web Workers) so that re-renders stay fast and the UI remains responsive.”
