# Performance Engineering (Frontend Focus)

This ties together the different layers from `01. Performance Optimization.md`.

---

## Levers You Control

- **Controlling unnecessary renders**
  - Code splitting & lazy loading
  - Memoization (`React.memo`, `useMemo`, `useCallback`)

- **Controlling unnecessary re-renders**
  - Stable props
  - Avoiding inline object/function creation
  - Proper key usage in lists

- **Controlling events & network**
  - Debouncing & throttling
  - Caching (React Query / SWR)
  - Request deduplication, pagination, prefetching, optimistic UI

---

## Frontend vs Backend Performance

- **Frontend**
  - Bundle size
  - Render cost
  - Web Vitals
  - Perceived performance

- **Backend**
  - Profiling
  - Memory analysis
  - CPU bottlenecks

---

## Interview Line

> “I treat performance as an engineering discipline: I measure (Web Vitals, DevTools), identify bottlenecks at bundle, render, and network layers, then apply techniques like memoization, code splitting, debouncing, and caching, and re-measure to confirm the improvements.”
