# Render Memoization (`React.memo`, `useMemo`, `useCallback`)

Memoization in React is about **avoiding unnecessary re-renders and recomputations**.

---

## 1. `React.memo`

Wraps a component so that it **only re-renders when its props change (shallow compare)**.

Good for:

- Pure presentational components
- Components that are expensive to render

---

## 2. `useMemo`

Caches the **result of an expensive computation** between renders.

```js
const value = useMemo(() => heavyCalculation(data), [data]);
```

Use when:

- The calculation is expensive
- Inputs don’t change on every render

---

## 3. `useCallback`

Caches a **function reference** so it doesn’t change on every render.

```js
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]);
```

Useful when:

- Passing callbacks to memoized children (`React.memo`)
- Preventing unnecessary re-renders due to changing function references

---

## How to Explain in Interview

> “At the render layer, I use React.memo, useMemo, and useCallback to avoid unnecessary re-renders and heavy recomputations, especially for expensive components and long lists.”
