# Bundle Analysis

## What it is

Bundle analysis means inspecting the final JavaScript bundle to understand:

- What is increasing the bundle size
- Which libraries are heavy
- Duplicate dependencies
- Unused code that is included

In simple terms:

> “We analyze what we are shipping to the browser.”

---

## Why it matters

Large bundles lead to:

- Slower initial load
- Poor performance on slow networks
- Worse Core Web Vitals (especially LCP)

---

## Tools (Mention Confidently in Interview)

- `webpack-bundle-analyzer`
- Vite bundle visualizer
- `next build --analyze`
- Chrome DevTools → **Coverage** tab

---

## How to Explain in Interview

You can say:

> “I use bundle analysis tools to identify large third-party libraries, unused code, and duplication. Based on the analysis, I optimize imports, remove unnecessary dependencies, and implement code splitting.”

That’s a strong and structured answer.
