# Concurrent Rendering in React

Concurrent features in React allow React to:
Start rendering, pause rendering, resume rendering, and even discard rendering work if needed.
It makes rendering interruptible.

Old React (legacy mode) was: Render → Commit → Done
Once render started, it blocked the main thread until finished.

## 🧠 Why Concurrent Rendering Was Introduced

When a component tree becomes large and rendering work takes significant time, the UI can suffer:

- UI freezes
- Typing feels laggy
- Scrolling stutters

This happens because traditional rendering was synchronous — React would block the main thread until rendering finished.

### ✅ What Concurrent Rendering Enables

Concurrent rendering allows React to:

- **Prioritize urgent updates** (e.g., typing in an input field)
- **Delay non-urgent updates** (e.g., rendering a large filtered list)
- **Keep the UI responsive** instead of blocking the browser

Instead of doing all rendering work in one uninterrupted block, React can:

- Break rendering into smaller units
- Pause work when needed
- Resume later
- Discard outdated renders if a newer update comes in

This improves perceived performance and user experience — especially in complex applications.

Concurrent rendering is enabled when you use:
createRoot(...) Instead of legacy ReactDOM.render.
React 18 enables concurrent features by default with createRoot.

🧠 What Problem Does It Solve?

- Input lag
- Heavy rendering blocks
- Better scheduling
- Smoother UX
