# 8 Patterns that cover about 80–90% of frontend interview coding questions (arrays, strings, Map, Set)

## 1. Hash Lookup Pattern

**Core DS:** `Set` or `Map`

**Use when:** Fast existence check (O(1))

```ts
Set<number>;
Map<number, number>;
```

---

## 2. Frequency Counting Pattern

**Core DS:** `Map<element, count>`

**Use when:** Counting occurrences

```ts
Map<string, number>;
```

---

## 3. Frequency Cancellation Pattern

**Core DS:** `Map<element, count>`

**Use when:** Comparing two datasets efficiently

👉 Same DS as frequency counting — just different usage (increment + decrement)

---

## 4. Grouping Pattern

**Core DS:** `Map<key, array[]>`

**Use when:** Categorizing/grouping items

```ts
Map<string, string[]>;
```

---

## 5. Two Pointer Pattern

**Core DS:** ❗ No extra DS (in-place)

**Use when:** Working on arrays (usually sorted)

👉 Only uses:

```ts
array + indices(left, right);
```

✔ Important:

- Space complexity → O(1)
- No Map / Set needed

---

## 6. Sliding Window Pattern

**Core DS:**

- Always: **array/string + pointers**
- Sometimes: `Set` or `Map`

### Basic Window (no DS)

```ts
number[]
```

### Advanced Window (with tracking)

```ts
Set<string>;
Map<string, number>;
```

✔ Rule:

- "unique / no repeat" → use `Set`
- "count / frequency" → use `Map`

---

## 7. Stack Pattern

**Core DS:** `Array (used as stack)`

```ts
string[]
number[]
```

👉 JS uses array as stack:

- `push()`
- `pop()`

---

## 8. Prefix Sum Pattern

**Core DS:** `Array`

```ts
number[]
```

👉 Sometimes combined with:

```ts
Map<number, number>; // advanced cases
```

---

# ⚠️ Two major Pattern Categories across DSA problems, (most of questions involve both: Traversal + Storage combined)

## 1. Storage-based patterns (DS heavy)

- Hash Lookup → Set / Map
- Frequency increment or decrement → Map
- Grouping → Map
- Stack → Array

---

## 2. Traversal-based patterns (logic heavy)

- Two Pointer → no DS
- Sliding Window → optional DS
- Prefix Sum → array (+ optional map)

---

# 🧠 Golden Rule

When DS is not mentioned:

| Pattern        | Default DS      |
| -------------- | --------------- |
| Two Pointer    | None (in-place) |
| Sliding Window | Maybe Set/Map   |
| Prefix Sum     | Array           |

---

# 🔥 Interview Thinking Rule

Ask:

👉 "Do I need memory or just movement?"

- Need memory → Map / Set
- Need movement → pointers
- Need history → stack
- Need accumulation → prefix array

---

# ❗ Important Clarification

❌ Missing DS ≠ any DS can be used  
✔ It means:

👉 The pattern is defined by **how you traverse**, not by storage
