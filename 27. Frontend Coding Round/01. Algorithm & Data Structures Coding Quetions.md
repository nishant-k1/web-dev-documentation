In algorithms and data-structure problems, most solutions fall into a small number of well-known patterns.
Developers may think they invented something new, but usually it’s a variation of an existing approach.

1. Frequency Counting Pattern (Map)
2. Frequency Decrement / Cancellation Pattern (Map)
3. Hash Lookup Pattern (Map / Set)
4. Two Pointers Pattern (Array/ String)
5. Sliding Window Pattern (substring or subarrays)
6. Grouping Pattern (Hash Grouping) (Map)
7. Stack Pattern (stack)
8. Prefix Sum Pattern (array)
9. Fast/Slow Pointer Pattern
10. Binary Search Pattern
11. Backtracking Pattern
12. Greedy Pattern

Almost every new idea is a variation of an existing pattern.

But programmers don’t memorize all algorithms. Instead they learn patterns. Once you know these patterns, new problems become recognizable.

For example:

```text
duplicate detection → Set lookup
group similar items → Map grouping
character counts → frequency map
substring problems → sliding window
```

Your brain eventually starts recognizing them automatically.

## Rules for patterns

1. Hash Lookup Pattern (Map / Set)
   **Rule**

   ```text
   Check first
   Store later
   ```

   Example:

   ```text
   Two Sum
   First duplicate
   Contains duplicate
   ```

2. Frequency Counting Pattern
   **Rule**

   ```text
   Increment while scanning
   ```

   Pattern:

   ```js
   map.set(key, (map.get(key) ?? 0) + 1);
   ```

   Example:

   ```text
   First non-repeating character
   Character frequency
   Duplicate counts
   ```

3. Frequency Cancellation Pattern
   **Rule**

   ```text
   Count from first dataset
   Cancel using second dataset
   ```

   Example:

   ```text
   Anagram check
   Ransom note
   Permutation comparison
   ```

4. Sliding Window Pattern
   **Rule**

   ```text
   Expand window
   Shrink window
   Track best result
   ```

   Mental model:

   ```text
   right pointer expands
   left pointer shrinks
   ```

   Example:

   ```text
   Longest substring without repeating characters
   Maximum sum subarray
   Minimum window substring
   ```

5. Two Pointer Pattern
   **Rule**

   ```text
   Pointers move toward solution
   ```

   Example structures:

   ```text
   left++
   right--
   ```

   or

   ```text
   slow++
   fast++
   ```

   Example:

   ```text
   Reverse array
   Pair sum in sorted array
   Remove duplicates
   ```

6. Grouping Pattern (Map → Array)
   **Rule**

   ```text
   Compute a key
   Group by that key
   ```

   structures:

   ```text
   Map<key, array>
   ```

   Example:

   ```text
   anagram groups → sorted word
   group by age → age
   group by remainder → num % k
   ```

7. Stack Pattern
   **Rule**

   ```text
   Push when opening
   Pop when closing
   ```

   Mental model::

   ```text
   Last in → first out
   ```

   Example:

   ```text
   Valid parentheses
   Next greater element
   Browser history
   ```

8. Prefix Sum Pattern
   **Rule**

   ```text
   Store cumulative sum
   Use previous sums to answer quickly
   ```

   Example:

   ```text
   Subarray sum equals k
   Range sum queries
   ```

9. Fast / Slow Pointer Pattern
   **Rule**

   ```text
   Move pointers at different speeds
   ```

   Example:

   ```text
   Detect cycle in linked list
   Find middle of linked list
   ```

## Why these rules matter

Interviewers expect you to think like this:

Instead of: `How do I solve this problem?`
You think: `Which pattern does this belong to?`

Then apply the rule.
Example:

```text
find duplicates → Hash lookup
count characters → Frequency map
substring problem → Sliding window
group items → Map grouping
```

Strong candidates often solve problems like this:

```text
Problem → recognize pattern → apply rule → code
```

Not:

```text
Problem → invent new algorithm
```


they think:

```text

```
