# 🎯 Frontend Interview Breakdown (JavaScript + DSA)

Frontend coding interviews usually test **four major areas**:

---

## 1. JavaScript Runtime & Language Behavior

These are **“what will be the output?”** questions.

They test whether you understand **how JavaScript actually executes code**.

### Topics

- Event loop
- Microtasks vs Macrotasks
- `setTimeout` / `setInterval`
- Promises
- `async / await`

### Example

```ts
console.log("A");

setTimeout(() => console.log("B"), 0);

Promise.resolve().then(() => console.log("C"));

console.log("D");
```

### This tests:

- Event loop
- Microtask queue
- Macrotask queue

---

## 2. JavaScript Core Language Concepts

These test **how JavaScript works internally**.

### Topics

- Closures
- Scope
- Hoisting
- Temporal Dead Zone (TDZ)
- `this` binding
- Prototype
- Type coercion
- Objects

---

### Example 1

```ts
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
```

### This tests:

- Closure
- Function scope (`var` vs `let`)
- Event loop

---

### Example 2

```ts
console.log([] + []);
console.log([] + {});
console.log({} + []);
```

### This tests:

- Type coercion
- Object → primitive conversion

---

## 3. Practical JavaScript Implementations

These are **real-world utilities** frontend developers are expected to build.

### Common Questions

- Debounce
- Throttle
- `Promise.all`
- `Promise.race`
- Deep clone
- Flatten array
- Flatten object
- call / apply / bind

---

### Example

```ts
debounce(fn, delay);
```

### This tests:

- Closures
- Timers
- Function references

---

## 4. Algorithm / Data Structure Questions

These are the **pattern-based problems**.

### Common Data Structures

- Array
- Map
- Set
- Stack

### Common Patterns

- Frequency counting
- Hash lookup
- Sliding window
- Two pointers
- Grouping

---

### Example Questions

- Two Sum
- Group Anagrams
- First Non-Repeating Character
- Longest Substring Without Repeating Characters
