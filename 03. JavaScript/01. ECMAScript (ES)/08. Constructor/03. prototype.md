# prototype

prototype is property of Constructor object which is also an object. This prototype object is used as blueprint for instances created from that constructor.
And all the methods and properties of Constructor object is actually the properties of the `.prototype` object which is inherited by the instances created from that constructor.

## Every Constructor Has a .prototype Object

- **`.prototype` property only exists for Constructor functions nothing else**

In JavaScript:

- Constructors are functions.
- Every function automatically has a `.prototype` property.
- That `.prototype` is an object.

So:

```js
typeof Array; // "function"
typeof Array.prototype; // "object"

typeof Object; // "function"
typeof Object.prototype; // "object"
```

Here,
**Array.prototype** is an object belonging to the Array constructor.
**Object.prototype** is an object belonging to the Object constructor.

## `.prototype` vs `__proto__`, vs `[[Prototype]]`

The .prototype property exists only on constructor functions. Regular objects do not have it. However, all objects have an internal [[Prototype]] link.

`.prototype` → lives on functions
`[[Prototype]]` → lives on objects

## Prototype chain (A chain like a linked list of objects where objects are linked via `[[Prototype]]`)

It behaves like a linked list, but it is not literally implemented as a JavaScript “LinkedList” data structure.

Instead:

Every object internally stores a hidden reference
The JavaScript engine manages this at the VM level
It’s part of the language runtime, not a userland data structure
So it’s not implemented as a JS “LinkedList” class — it’s a low-level engine mechanism.

```text
instance
  ↓
[[Prototype]]
  ↓
SomePrototypeObject
  ↓
[[Prototype]]
  ↓
Object.prototype
  ↓
[[Prototype]]
  ↓
null
```

✅ 1️⃣ Prototype chain = link among objects

The prototype chain is a chain of objects linked together.
And the linking happens via: `[[Prototype]]`

✅ 2️⃣ In constructor-based creation
When you do:

```js
function Person() {}
const p = new Person();
```

What actually happens is:

```text
p.[[Prototype]] → Person.prototype
```

So yes:

- Person.prototype is just a normal object
- It stores shared methods and properties

It is not special in nature — just an object.

## About `[[Prototype]]`

[[Prototype]]:

- Is NOT a property you can see directly
- Is NOT an object
- Is NOT something that contains methods
- Is just an internal reference (a pointer)

It simply stores:
“Who is my parent object?”
It does not contain logic.
It does not contain methods.
It only points.

It is simply a pointer in the linked list.

Every normal object has:

```js
object
  ↓
[[Prototype]]  (internal pointer)
  ↓
another object
```

And property lookup works like this:

- Check current object
- If not found → follow [[Prototype]]
- Repeat until null

That upward walking is the prototype chain.

So,

- The prototype chain is a chain of objects linked through the internal [[Prototype]] reference.
- In constructor-based creation, Constructor.prototype is just a normal object placed in that chain to provide shared properties.
- The [[Prototype]] itself is only a pointer to the next object in the chain; it does not contain properties or methods.
