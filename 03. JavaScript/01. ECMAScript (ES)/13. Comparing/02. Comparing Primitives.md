# Comparing Primitives

In JavaScript, primitive values are compared using strict equality (`===`) or loose equality (`==`) operators.

## Strict Equality (`===`)

- Compares both value and type.
- No type coercion occurs.
- Use this for precise, type-safe comparisons.

```js
5 === 5; // true
5 === "5"; // false (different types)
true === true; // true
null === null; // true
```

## Loose Equality (`==`)

- Compares value only, performing type coercion if the types differ.
- May lead to unexpected results due to type conversions.

```js
5 == 5; // true
5 == "5"; // true (type coercion)
null == undefined; // true (special case)
```

## Comparison of Specific Primitives

### 1. Strings

- Compared character by character (case-sensitive).

```js
"hello" === "hello"; // true
"Hello" === "hello"; // false
```

### 2. Numbers

- Compared by value.
- Special cases:
  - `NaN` is not equal to itself.
  - Use `Number.isNaN()` to check for `NaN`.

```js
5 === 5; // true
NaN === NaN; // false
Number.isNaN(NaN); // true
```

### 3. Booleans

- Compared by value.

```js
true === true; // true
false === false; // true
```

### 4. null and undefined

- `null` is only strictly equal to `null`
- `undefined` is only strictly equal to `undefined`
- `null` and `undefined` are loosely equal.

```js
null === null; // true
undefined === undefined; // true
null === undefined; // false
null == undefined; // true (loose equality)
```

### 5. Symbols

- Compared by reference, not value.
- Two Symbol instances with the same description are not equal.

```js
const sym1 = Symbol("desc");
const sym2 = Symbol("desc");
sym1 === sym2; // false (different references)
```

### 6. BigInt

- Compared by value if both are BigInt.

```js
5n === 5n; // true
5n === 5; // false (different types)
```

## Best Practices

1. **Always use `===`** - Prefer strict equality for type-safe comparisons
2. **Avoid `==`** - Loose equality can lead to unexpected behavior
3. **Use `Number.isNaN()`** - To check for NaN values
4. **Understand type coercion** - Know when and how JavaScript converts types
