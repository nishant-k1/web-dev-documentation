# Object literals

Are Object Literals Instances of Object constructor

When you create:

```js
const obj = {};
```

JavaScript internally does something conceptually similar to:

```js
const obj = new Object();
```

Both create an object whose prototype is:

```js
Object.prototype;
```

So

```js
obj instanceof Object; // true
```

Under the Hood (Prototype View)

```js
const obj = {};
console.log(Object.getPrototypeOf(obj) === Object.prototype); // true
```

**Even though object literals are instances of Object, they are:**

- Not created using the Object constructor explicitly
- Created using literal syntax
- More efficient and preferred in modern JavaScript

So instead of:

```js
const obj = new Object();
obj.name = "Nishant";
```

We use:

```js
const obj = { name: "Nishant" };
```
