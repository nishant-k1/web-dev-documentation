## Quick Summary

### Pass By Value (Primitives)

- A **copy** of the value is passed
- Changes to parameter **don't affect** original
- Applies to: `number`, `string`, `boolean`, `undefined`, `null`, `symbol`, `bigint`

```javascript
function modifyValue(x) {
  x = 42; // Only affects local copy
}

let num = 10;
modifyValue(num);
console.log(num); // 10 (unchanged)
```

### Pass By Reference (Non-Primitives)

- A **reference** to the object is passed
- Changes to object properties **affect** original
- Reassigning parameter **doesn't affect** original variable
- Applies to: `object`, `array`, `function`, `Date`, `RegExp`, etc.

```javascript
function modifyObject(obj) {
  obj.a = 42; // Modifies original object
}

let myObj = { a: 10 };
modifyObject(myObj);
console.log(myObj.a); // 42 (modified)
```

### Important: Reassignment Doesn't Work

```javascript
function reassign(obj) {
  obj = { a: 100 }; // Doesn't affect original
}

let myObj = { a: 10 };
reassign(myObj);
console.log(myObj.a); // 10 (unchanged)
```
