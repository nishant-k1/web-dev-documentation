# âœ… Must-Know `Number` Object

The `Number` object provides utilities for working with numbers and contains important constants and static methods.

---

## Number Primitive vs Number Object

### Primitive (Recommended)

```js
const num = 42; // Primitive number
typeof num; // "number"
```

### Number Object (Not Recommended)

```js
const numObj = new Number(42); // Number object
typeof numObj; // "object"
```

**Important:** Always use primitives, not Number objects!

## Number Constants

### Important Constants

```js
Number.MAX_VALUE; // 1.7976931348623157e+308 (largest representable number)
Number.MIN_VALUE; // 5e-324 (smallest positive number)
Number.MAX_SAFE_INTEGER; // 9007199254740991 (2^53 - 1)
Number.MIN_SAFE_INTEGER; // -9007199254740991
Number.POSITIVE_INFINITY; // Infinity
Number.NEGATIVE_INFINITY; // -Infinity
Number.NaN; // NaN
Number.EPSILON; // 2.220446049250313e-16 (smallest difference between numbers)
```

### Usage Examples

```js
// Safe integer range
if (num <= Number.MAX_SAFE_INTEGER && num >= Number.MIN_SAFE_INTEGER) {
  // Safe to use as integer
}

// Floating point precision
const difference = Math.abs(0.1 + 0.2 - 0.3);
if (difference < Number.EPSILON) {
  // Numbers are approximately equal
}
```

## Number Static Methods

### `Number.isNaN()` â­â­â­

Better than global `isNaN()` - doesn't coerce.

```js
Number.isNaN(NaN); // true
Number.isNaN("NaN"); // false (doesn't coerce)
Number.isNaN(undefined); // false

// vs global isNaN()
isNaN("NaN"); // true (coerces string to NaN)
```

### `Number.isFinite()` â­â­â­

Checks if value is finite number (not Infinity, -Infinity, or NaN).

```js
Number.isFinite(42); // true
Number.isFinite(Infinity); // false
Number.isFinite(-Infinity); // false
Number.isFinite(NaN); // false
Number.isFinite("42"); // false (doesn't coerce)

// vs global isFinite()
isFinite("42"); // true (coerces)
```

### `Number.isInteger()` â­â­â­

```js
Number.isInteger(42); // true
Number.isInteger(42.0); // true
Number.isInteger(42.1); // false
Number.isInteger(NaN); // false
Number.isInteger(Infinity); // false
```

### `Number.isSafeInteger()` â­â­â­

Checks if integer is within safe range.

```js
Number.isSafeInteger(42); // true
Number.isSafeInteger(9007199254740991); // true (MAX_SAFE_INTEGER)
Number.isSafeInteger(9007199254740992); // false (unsafe)
Number.isSafeInteger(0.5); // false (not integer)
```

### `Number.parseInt()` â­â­

Same as global `parseInt()`.

```js
Number.parseInt("42"); // 42
Number.parseInt("42px"); // 42
Number.parseInt("1010", 2); // 10 (binary)
Number.parseInt("FF", 16); // 255 (hexadecimal)
```

### `Number.parseFloat()` â­â­

Same as global `parseFloat()`.

```js
Number.parseFloat("42.5"); // 42.5
Number.parseFloat("42.5px"); // 42.5
Number.parseFloat("3.14e2"); // 314
```

## Number Instance Methods

### `toFixed(digits)` â­â­â­

Formats number with fixed decimal places.

```js
const num = 3.14159;

num.toFixed(2); // "3.14" (returns string!)
num.toFixed(0); // "3"
(0.1 + 0.2).toFixed(2); // "0.30"

// Rounding
(3.456).toFixed(2); // "3.46" (rounded)
```

### `toPrecision(precision)` â­â­

Formats number with specified precision.

```js
const num = 123.456;

num.toPrecision(5); // "123.46" (5 significant digits)
num.toPrecision(2); // "1.2e+2"
(0.1 + 0.2).toPrecision(2); // "0.30"
```

### `toExponential(fractionDigits?)` â­

Formats number in exponential notation.

```js
const num = 12345;

num.toExponential(); // "1.2345e+4"
num.toExponential(2); // "1.23e+4"
```

### `toString(radix?)` â­

Converts number to string, optionally with radix.

```js
const num = 255;

num.toString(); // "255"
num.toString(2); // "11111111" (binary)
num.toString(8); // "377" (octal)
num.toString(16); // "ff" (hexadecimal)
```

### `valueOf()`

Returns primitive value.

```js
const numObj = new Number(42);
numObj.valueOf(); // 42 (primitive)
```

## Common Use Cases

### 1. Safe Integer Checks

```js
function safeAdd(a, b) {
  if (
    Number.isSafeInteger(a) &&
    Number.isSafeInteger(b) &&
    Number.isSafeInteger(a + b)
  ) {
    return a + b;
  }
  throw new Error("Unsafe integer operation");
}
```

### 2. Floating Point Comparison

```js
function almostEqual(a, b, epsilon = Number.EPSILON) {
  return Math.abs(a - b) < epsilon;
}

almostEqual(0.1 + 0.2, 0.3); // true
```

### 3. Number Formatting

```js
// Currency formatting
function formatCurrency(amount) {
  return "$" + amount.toFixed(2);
}

formatCurrency(123.456); // "$123.46"

// Percentage
function formatPercent(value) {
  return (value * 100).toFixed(2) + "%";
}
```

### 4. Integer Validation

```js
function validateInteger(value) {
  if (!Number.isInteger(value)) {
    throw new TypeError("Value must be an integer");
  }
  return value;
}
```

## Important Notes

### âŒ Don't Use `new Number()`

```js
// Bad
const numObj = new Number(42);

// Good
const num = 42; // Use primitive
```

### âš ï¸ `toFixed()` Returns String

```js
const num = 3.14159;
const formatted = num.toFixed(2); // "3.14" (string, not number!)

// Convert back to number if needed
const numAgain = parseFloat(formatted); // 3.14
```

### âš ï¸ Precision Loss

```js
0.1 + 0.2; // 0.30000000000000004 (not exactly 0.3!)

// Use epsilon for comparison

```

## Interview Common Questions

### Q: Difference between `Number.isNaN()` and `isNaN()`?

**Number.isNaN():**

- Doesn't coerce
- Only returns true for actual NaN

**isNaN():**

- Coerces value to number first
- Can give false positives

```js
Number.isNaN("NaN"); // false
isNaN("NaN"); // true (coerces to NaN)
```

### Q: What is `Number.MAX_SAFE_INTEGER`?

The largest integer that can be safely represented in JavaScript (2^53 - 1). Integers larger than this may lose precision.

### Q: How to check if a number is safe?

```js
Number.isSafeInteger(value);
```

### Q: How to format a number with 2 decimal places?

```js
num.toFixed(2); // Returns string "3.14"
```

### Q: How to convert number to binary/hex?

```js
num.toString(2); // Binary
num.toString(16); // Hexadecimal
```
