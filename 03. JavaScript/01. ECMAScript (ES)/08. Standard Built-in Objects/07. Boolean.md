# ‚úÖ Must-Know `Boolean` Object

The `Boolean` object is a wrapper for boolean primitive values. It's rarely used directly, but understanding it is important.

---

## Boolean Primitive vs Boolean Object

### Primitive (Recommended)

```js
const bool = true; // Primitive boolean
typeof bool; // "boolean"
```

### Boolean Object (Not Recommended)

```js
const boolObj = new Boolean(true); // Boolean object
typeof boolObj; // "object"
```

**Important:** Always use primitives, not Boolean objects!

---

## Boolean Constructor

### Creating Boolean

```js
// Primitive (recommended)
const bool = true;
const bool2 = false;
const bool3 = Boolean(1); // true (converts to boolean)

// Object (not recommended)
const boolObj = new Boolean(true); // Avoid!
```

### Type Conversion

```js
// Truthy values ‚Üí true
Boolean(1); // true
Boolean("hello"); // true
Boolean([]); // true
Boolean({}); // true

// Falsy values ‚Üí false
Boolean(0); // false
Boolean(""); // false
Boolean(null); // false
Boolean(undefined); // false
Boolean(NaN); // false
Boolean(false); // false
```

## Boolean Instance Methods

### `toString()`

```js
const bool = true;
bool.toString(); // "true"
false.toString(); // "false"

// With Boolean object
const boolObj = new Boolean(true);
boolObj.toString(); // "true"
```

### `valueOf()`

```js
const boolObj = new Boolean(true);
boolObj.valueOf(); // true (primitive value)
```

## Common Use Cases

### 1. Explicit Type Conversion

```js
// Convert value to boolean
function toBoolean(value) {
  return Boolean(value);
}

toBoolean(1); // true
toBoolean(0); // false
toBoolean(""); // false
```

### 2. Boolean Filtering

```js
const arr = [1, 0, "", "hello", null, undefined];
const truthy = arr.filter(Boolean); // [1, "hello"]
```

### 3. Double Negation (Common Pattern)

```js
// Convert to boolean using !!
const value = "hello";
const bool = !!value; // true

// Equivalent to Boolean(value)
!!0; // false
!!""; // false
!![]; // true
```

## Important Notes

### ‚ùå Don't Use `new Boolean()`

```js
// Bad
const boolObj = new Boolean(true);

// Problem: Object is always truthy!
if (boolObj) {
  console.log("This always runs, even if value is false!");
}

// Good
const bool = true; // Use primitive
```

### ‚úÖ Always Use Primitives

```js
// Primitives work correctly
const isActive = true;
if (isActive) {
  console.log("Active");
}

// Boolean objects don't work as expected
const isActiveObj = new Boolean(false);
if (isActiveObj) {
  console.log("This runs even though value is false!");
}
```

## Interview Common Questions

### Q: Difference between `Boolean()` and `new Boolean()`?

- `Boolean(value)`: Converts value to primitive boolean
- `new Boolean(value)`: Creates Boolean object (avoid)

### Q: How to convert a value to boolean?

```js
Boolean(value); // Explicit conversion
!!value; // Double negation (common pattern)
```

### Q: What are falsy values in JavaScript?

```js
Boolean(false); // false
Boolean(0); // false
Boolean(-0); // false
Boolean(0n); // false (BigInt zero)
Boolean(""); // false
Boolean(null); // false
Boolean(undefined); // false
Boolean(NaN); // false
```

Everything else is truthy!
