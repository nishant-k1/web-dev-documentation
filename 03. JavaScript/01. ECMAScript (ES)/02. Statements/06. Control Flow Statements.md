# Control Flow Statements

- Control the flow of program execution.
- In control flow statements, the expression used as condition is first evaluated to a value, and then that value is coerced to boolean to decide execution.
- In control flow statements, the evaluated result of the expression used as condition is always coerced to a boolean to determine whether the block executes.

**NOTE**: A “condition” is not a special type of expression.
It’s any expression used in a place where a boolean is expected.
Any expression placed inside a control-flow construct (like if, while, etc.) is used as a condition.

Any expression used as condition always coerces to boolean.

**Expression**: Anything that produces a value
**Condition**: An expression whose value is interpreted as true/false

There is:

❌ No special “condition expression” type in JavaScript
✔ Just normal expressions used in a boolean context

A condition in JavaScript is any expression whose result is evaluated as a boolean in control flow statements.

## 1. Conditional Statements

### `if` block statement

```js
if (condition) {
  // code
}
```

Here, the expression used as condition is first evaluated which returns some value (not necessarily boolean) then implicitly that value is coerced to boolean.

Runs only when:

1. Its condition is true

### `else if` block statement

```js
if (condition1) {
  // code
} else if (condition2) {
  // code
} else {
  // code
}
```

Runs only when:

1. ALL previous conditions are false  
   AND
2. Its own condition is true

### `else` block statement

```js
if (condition) {
  // code
} else {
  // code
}
```

Runs only when:

1. ALL previous conditions are false

### Multiple independent if blocks

```js
if (cond1) {
  /* runs if cond1 true */
}
if (cond2) {
  /* runs if cond2 true */
}

if (cond3) {
  /* runs if cond2 true */
}
```

- Each `if` is evaluated independently
- All conditions are checked
- Multiple blocks **can run**

### Nested if blocks

```js
if (cond1) {
  if (cond2) {
    // runs only if BOTH cond1 and cond2 are true
  }
}
```

Runs only when:

1. The outer condition is true
2. Then the inner condition is true

### if else summary

- `if / else if / else` → **choose ONE path**
- Multiple `if` → **check ALL paths**
- Nested `if` → **condition inside a chosen path**

### switch statement

```js
switch (condition) {
  case 1:
    // statement 1
    // statement 2
    // statement 3
    break;
  case 2:
    // statement 1
    // statement 2
    // statement 3
    break;
  default:
  // code
}
```

- cases are values, not condition, so cases should always be unique
- There's no block using curly braces for the cases, yet we can write multiple lines of statements inside the case. A case is not a block—it’s a label, and everything after it runs until a `break` (or next case)
- switch is not "find the matching case and run only that". It is actually "find the matching case and run everything from there to the end". To prevent the all down test cases we need to use `break`. break is what makes it behave the way you intuitively expect — run only the matching case and stop.

- Step-by-step execution flow:
  - Finds first matching case
  - Jumps there
  - Executes everything below
  - Stops at break statement and exits the switch completely not just a case.
    - With break → “Stop now”
    - Without break → “Keep going until nothing left”
  - return statement immediately exits the entire function, not just the switch. return only works inside functions.
  - default: Runs only if No case matched OR fall-through reaches it.

## 2. Loop Statements

### for loop

```js
for (let i = 0; i < 10; i++) {
  // code
}
```

### while loop

```js
while (condition) {
  // code
}
```

### do-while loop

```js
do {
  // code
} while (condition);
```

### for...in loop

Iterates over object properties.

```js
for (const key in object) {
  // code
}
```

### for...of loop

Iterates over iterable values (arrays, strings, etc.).

```js
for (const value of array) {
  // code
}
```

### for-await-of loop

Iterates over async iterables (async generators, async iterators). Introduced in ES2018.

```js
for await (let value of asyncIterable) {
  // code
}
```

**Key characteristics:**

- Works with async iterables (objects with `Symbol.asyncIterator`)
- Can only be used inside async functions or at the top level of modules
- Awaits each value as it's yielded
- Used with async generators, async iterators, and other async iterables

**Examples:**

```js
// With async generator
async function* asyncNumbers() {
  for (let i = 0; i < 5; i++) {
    await new Promise((resolve) => setTimeout(resolve, 100));
    yield i;
  }
}

async function processNumbers() {
  for await (const num of asyncNumbers()) {
    console.log(num); // 0, 1, 2, 3, 4 (with delays)
  }
}

// With async iterable
async function* fetchPages() {
  for (let i = 1; i <= 3; i++) {
    const response = await fetch(`/api/page/${i}`);
    yield response.json();
  }
}

async function displayPages() {
  for await (const page of fetchPages()) {
    console.log(page);
  }
}
```

## 3. Loop Control Statements

Control the execution of loops.

### break

Exits the loop immediately.

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) break; // exits loop when i is 5
}
```

### continue

Skips to the next iteration.

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) continue; // skips iteration when i is 5
}
```
