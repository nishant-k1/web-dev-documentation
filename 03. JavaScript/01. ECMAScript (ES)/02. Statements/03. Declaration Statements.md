# Declaration Statements

Declaration statements create new bindings (variables, functions, classes) in the current scope.

## Key Characteristics

- **Create bindings** - All declarations create new identifiers in the scope
- **Hoisting behavior** - Different declarations have different hoisting behaviors
- **Scope** - Declarations are scoped (function-scoped, block-scoped, or module-scoped)
- **Cannot be redeclared** - Most declarations cannot be redeclared in the same scope (except `var`)

## Types of Declaration Statements

1. Variable Declarations (`var`, `let`, `const`)
2. Function Declarations (`function`)
3. Class Declarations (`class`)
4. Generator Function Declarations (`function*`)
5. Async Function Declarations (`async function`)
6. Async Generator Function Declarations (`async function*`)

## Important Notes

1. **Hoisting**: Function and `var` declarations are hoisted; `let`, `const`, and `class` are not
2. **Temporal Dead Zone (TDZ)**: `let`, `const`, and `class` are in TDZ until declaration is reached
3. **Redeclaration**: `var` can be redeclared; `let`, `const`, `function`, and `class` cannot (in same scope)
4. **Scope**: `var` is function-scoped; `let`, `const`, `function`, and `class` are block-scoped
5. **Best Practices**: Prefer `const` by default, use `let` when reassignment is needed, avoid `var` in modern JavaScript
