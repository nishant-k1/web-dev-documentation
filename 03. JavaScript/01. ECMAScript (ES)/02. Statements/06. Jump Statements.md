# Jump Statements

Transfer control to another part of the program.

**Note:** Some statements like `break`, `continue`, and `throw` appear in multiple categories because they serve multiple purposes. They're covered in detail in their primary sections (Loop Control and Exception Handling) and referenced here for completeness.

## return

Exits function and returns a value.

```js
function add(a, b) {
  return a + b; // exits function and returns value
}

function greet(name) {
  if (!name) {
    return; // exits early, returns undefined
  }
  return `Hello, ${name}!`;
}
```

## break

Exits loop or switch (can also be used with labels).

```js
// In switch
switch (x) {
  case 1:
    break; // exits switch
}

// In loop
for (let i = 0; i < 10; i++) {
  if (i === 5) break; // exits loop
}

// With label
outerLoop: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outerLoop; // breaks out of outer loop
  }
}
```

## continue

Continues to next loop iteration (can also be used with labels).

```js
// In loop
for (let i = 0; i < 10; i++) {
  if (i % 2 === 0) continue; // skip even numbers
  console.log(i);
}

// With label
outerLoop: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (j === 2) continue outerLoop; // continues outer loop
  }
}
```

## throw

Throws an exception (covered in detail in [Exception Handling Statements](./05.%20Exception%20Handling%20Statements.md)).

```js
throw new Error("Something went wrong");
throw "String error"; // Can throw any value
```

## Important Notes

- **return** can only be used inside functions
- **break** and **continue** are primarily used in loops and switches
- **throw** transfers control to the nearest exception handler
- All jump statements interrupt the normal flow of execution
