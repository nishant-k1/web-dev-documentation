# With Statement (Deprecated)

The `with` statement extends the scope chain. **Deprecated** and should not be used.

## Syntax

```js
with (object) {
  // statements
}
```

## Example

```js
// ⚠️ Deprecated - do not use
with (object) {
  // properties of object can be accessed directly
  property = value; // equivalent to object.property = value
  method(); // equivalent to object.method()
}
```

## Why It's Deprecated

1. **Performance issues** - Makes code slower due to dynamic scope lookup
2. **Strict mode** - Not allowed in strict mode (throws SyntaxError)
3. **Ambiguity** - Makes it unclear where variables come from
4. **Optimization problems** - Prevents JavaScript engines from optimizing code
5. **Maintenance issues** - Makes code harder to understand and maintain

## What It Does

The `with` statement adds the properties of an object to the scope chain, allowing you to access them without the object prefix.

```js
// Without with
let obj = { x: 1, y: 2 };
console.log(obj.x); // 1
console.log(obj.y); // 2

// With with (DON'T USE)
with (obj) {
  console.log(x); // 1 (accesses obj.x)
  console.log(y); // 2 (accesses obj.y)
}
```

## Problems

### 1. Ambiguity

```js
let x = 10;
let obj = { x: 20 };

with (obj) {
  console.log(x); // Which x? obj.x (20) or outer x (10)?
  // JavaScript checks obj first, then outer scope
}
```

### 2. Performance

```js
// Slow - JavaScript must check object properties at runtime
with (obj) {
  property1 = value1;
  property2 = value2;
  // Each access requires dynamic lookup
}

// Fast - Direct property access
obj.property1 = value1;
obj.property2 = value2;
```

### 3. Strict Mode Error

```js
"use strict";

with (obj) {
  // ❌ SyntaxError: Strict mode code may not include a with statement
  // code
}
```

## Alternatives

### Use Object Destructuring

```js
// Instead of with
let obj = { x: 1, y: 2, z: 3 };

// ✅ Use destructuring
let { x, y, z } = obj;
console.log(x, y, z); // 1, 2, 3
```

### Use Direct Property Access

```js
// ✅ Clear and explicit
obj.x = 1;
obj.y = 2;
obj.method();
```

### Use a Helper Function

```js
// ✅ Create a helper if you need repeated access
function withObject(obj, callback) {
  return callback(obj);
}

withObject(obj, (o) => {
  console.log(o.x, o.y);
});
```

## Important Notes

- **Never use in new code** - Always deprecated
- **Strict mode** - Throws SyntaxError in strict mode
- **Modern alternative** - Use object destructuring or direct property access
- **Legacy code** - May exist in old codebases, should be refactored
