# Inheritance in Constructor

For constructor-based objects:
Shared instance methods must live on `.prototype`, not on the constructor function itself.
Properties and methods inherited by instances live on `.prototype`, not on the constructor function itself.

**The constructor function is just:**

- A factory
- An initializer

**The .prototype object is:**

- The shared method container
- The blueprint for instances

## Where Do Inherited Methods Actually Live

When you write:

```js
function Person(name) {
  this.name = name;
}

Person.prototype.greet = function () {
  return "Hi " + this.name;
};

const p1 = new Person("Nishant");
```

The structure is:

```text
  p1
  ↓
Person.prototype   ← greet() lives here
  ↓
Object.prototype
  ↓
null
```

So:

- greet is stored on Person.prototype
- p1 inherits it via the prototype chain
- Person (the constructor function itself) does NOT provide it directly to instances

## What If You Put a Method on the Constructor Itself?

```js
Person.sayHello = function () {
  return "Hello";
};
```

Now:

- Person.sayHello() works
- But p1.sayHello() does NOT work

Because:

```text
Person          ← sayHello lives here
  ↓
Function.prototype
  ↓
Object.prototype
  ↓
null
```

That chain belongs to the function object, not to p1.

## Important Clarification

This is true specifically for constructor/class-based creation.

More generally:

- Inherited properties always live on whatever object is in the prototype chain — not necessarily on a constructor’s prototype.

Example:

```js
const base = { greet() {} };
const obj = Object.create(base);
```

Here:

- greet lives on base
- No constructor involved
- Still inherited via prototype chain

## so, what if the instance is not created from constructor and created from instance? then what happens which properties gets shared because instance itself don't have prototype property

- `.prototype` Is NOT Required for Sharing
- Property sharing does NOT depend on .prototype.
- It depends on the internal [[Prototype]] link.
- `.prototype` is only a convenience property on constructor functions.

### Example: Instance Created From Instance

```js
const base = {
  greet() {
    return "Hello";
  },
};

const obj = Object.create(base);
```

Here:

- base is just a normal object (an “instance” in your words).
- obj is created from base.

Now the chain:

```text
obj
  ↓
base
  ↓
Object.prototype
  ↓
null
```

So:

- greet lives on base
- obj inherits it via prototype chain

No constructor involved.
No .prototype involved.

🔹 Key Insight

instance itself doesn’t have .prototype. That is correct but it does not need to.

Because:

Every object has an internal slot:

`[[Prototype]]`

When you do:

`Object.create(base)`

JavaScript does:

`newObj.[[Prototype]] = base`

**Sharing works because of `[[Prototype]]` linkage — not because of `.prototype`.**

Everything that exists on the prototype object (in this case base) is accessible to obj unless shadowed.

```js
obj.greet(); // inherited
```

If you later add:

```js
base.sayBye = function () {
  return "Bye";
};
```

Then:

```js
obj.sayBye(); // works immediately
```

Because they share the same prototype object.

## The Universal Rule (Very Important)

Inheritance in JavaScript depends ONLY on:
`object.[[Prototype]]`

- Not on constructors.
- Not on .prototype.
- Not on classes.

## If an object is created from another object (instance)

- The new object’s [[Prototype]] points to that instance.
- All properties on that instance become inherited.
- No constructor or .prototype is required.

## 🔹 Clean Mental Model

There are two different things:

1. **.prototype**
   - Exists only on functions
   - Used by new

2. **[[Prototype]]**
   - Exists on every object
   - Controls inheritance

## then `[[Prototype]]` must be existing on `.prototype` property of the constructors?

**Yes. Because `.prototype` is just a normal object — and all objects have `[[Prototype]]`.**

🔹 Step 1: What Is .prototype?

When you write:

```js
function Person() {}
```

JavaScript automatically creates:

```js
Person.prototype;
```

This is just a normal object.

You can verify:

```js
typeof Person.prototype; // "object"
```

🔹 Step 2: Do All Objects Have [[Prototype]]?

Yes.
Every object (except those created with Object.create(null)) has an internal:
So:

```js
Object.getPrototypeOf(Person.prototype);
```

returns:

```js
Object.prototype;
```

So the structure is:

```text
Person.prototype
  ↓
Object.prototype
  ↓
null
```
