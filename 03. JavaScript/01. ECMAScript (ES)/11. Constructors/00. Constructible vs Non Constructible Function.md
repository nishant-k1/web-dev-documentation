# Constructor Function

1. **A Function Definition Alone Is Just a Function**

   When you write:

   ```js
   function exampleFun() {}
   ```

   At this point:
   1. It is just a function.
   2. It is not yet acting as a constructor.
   3. Nothing has been constructed.

   Two things exist:
   1. The constructor function (Person)
   2. The prototype object (Person.prototype)

2. **But Can It Be Used As a Constructor?**

   That depends on whether the function is **constructible**.
   Normal function declarations and expressions:

   ```js
   function exampleFun() {}
   ```

   1. Have `.prototype`
   2. Have internal `[[Construct]]`
   3. Can be used with `new`

   So they are **constructible functions**.
   Meaning: 4. They have the capability to act as constructors.

3. **So Is It a Constructor or Not?**

   Here’s the precise answer:
   1. Technically: It is a constructible function.
   2. Practically: It becomes a constructor when used with `new`.
   3. By convention: We call it a “constructor function” if it is intended to be used with `new`.

   So the definition alone does not “make” it a constructor in execution — but it is capable of being one.

## 4. Compare Different Function Types

### 4.1 Normal Function

```js
function A() {}
```

1. Has `.prototype`
2. Can be used with `new`
3. Has `[[Construct]]`
4. Is constructible

### 4.2 Arrow Function

```js
const A = () => {};
```

1. No `.prototype`
2. No `[[Construct]]`
3. Cannot be used with `new`

So this one is **not constructible at all**.

### 4.2 Method Shorthand

It is a special syntax used:

1. Inside object literals

   ```js
   const obj = { greet() {} };
   console.log(obj.greet.prototype); // ❌ undefined
   ```

2. Inside classes

   ```js
   class User {
     greet() {}
   }
   ```

Both are “method definitions”. Why Is It Special?

1. No `.prototype`
2. No `[[Construct]]`
3. Cannot be used with `new`

So this one is **not constructible at all**.

Why Did JavaScript Do This?

Because methods are meant to be: Behavior of an object — not constructors. When you define:

```js
const obj = {
  greet() {},
};
```

You’re saying:
“greet is a behavior of this object.”

Not:
“greet is a blueprint for creating new objects.”
So the engine removes constructor capability.

## 5. Very Precise Terminology

There are 3 levels here:

1. Function
2. Constructible function (has `[[Construct]]`)
3. Constructor call (when invoked with `new`)

Your function definition:

```js
function exampleFun() {}
```

1. It is level 2.
2. It is a constructible function.

## 6. Think of It Like This

The function definition gives the function:

1. A `.prototype`
2. The ability to be called with `new`

But until `new` is used, it hasn’t constructed anything.
