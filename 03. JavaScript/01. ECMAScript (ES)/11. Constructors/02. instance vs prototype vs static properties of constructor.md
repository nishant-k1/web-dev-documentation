# Constructors

A constructors have 3 different types properties based on where they live inside the constructor: 1. `instance` 2. `prototype` 3. `static`

1. **Inside constructor → instance properties**

   ```js
   function Car() {
     this.brand = "BMW";
     this.drive = function () {
       console.log("Driving");
     };
     const model = "";
   }

   const c = new Car();
   const c1 = new Car();
   const c2 = new Car();
   c1.drive(); // ✅
   c2.drive(); // ✅
   c1.drive() === c2.drive(); //  ❌
   c.brand; // ✅
   c.drive(); // ✅
   Car.brand; // ❌
   Car.drive; // ❌
   ```

   👉 Each instance gets a new reference for those properties (brand)
   👉 Not copied → created again each time
   👉 Accessible from instances
   👉 Not shared (separate copies)
   👉 Instance property overrides (shadows) prototype property

   👉 Variables declared with const, let, or var inside a constructor are:

   ✅ Just local variables (function-scoped or block-scoped)
   c.model; // ❌
   Car.model; // ❌

   👉 They are NOT:

   ❌ instance properties
   ❌ prototype properties
   ❌ static properties
   Not on instance
   Not on prototype
   But still accessible via closure ✅

2. **On prototype → shared properties**
   Shared = one storage location reused by all instances

   ```js
   function Car() {}
   Car.prototype.drive = function () {
     console.log("Driving");
   };

   const c1 = new Car();
   const c2 = new Car();
   c1.drive(); // ✅
   c2.drive(); // ✅
   c1.drive() === c2.drive(); // ✅

   Car.prototype.drive; // ✅
   Car.drive; // ❌
   ```

   👉 Accessible from instances
   👉 Shared (one underlying value)

3. **On constructor itself → static properties**

   ```js
   function Car() {}
   Car.type = "Vehicle";

   const c = new Car();

   Car.type; // ✅
   c.type; // ❌
   ```

| Type      | Where defined    | Accessed via instance? | Shared? | Purpose           |
| --------- | ---------------- | ---------------------- | ------- | ----------------- |
| Instance  | `this.x`         | ✅                     | ❌      | Per-object data   |
| Prototype | `Fn.prototype.x` | ✅                     | ✅      | Shared behavior   |
| Static    | `Fn.x`           | ❌                     | ❌      | Utility / factory |

## on prototype properties(shared properties) and the instance properties (Inside constructor) both accessible from instance so what's the difference among all?

Both instance and prototype are accessible from instances, but only prototype properties are shared because they live in one place instead of being copied per object.
