# Template Literals Overview

Template literals are string literals allowing embedded expressions. They were introduced in ES6 and provide enhanced string functionality over traditional string concatenation.

## Basic Syntax

```javascript
const name = "World";
const greeting = `Hello, ${name}!`; // "Hello, World!"

// Multi-line strings
const multiLine = `Line 1
Line 2
Line 3`;

// Expression interpolation
const sum = `2 + 2 = ${2 + 2}`; // "2 + 2 = 4"
```

## Tagged Templates

Tagged templates allow you to parse template literals with a function. The first argument is an array of string values, and subsequent arguments are the interpolated expressions.

```javascript
function tag(strings, ...values) {
  // Process template literal
  return strings[0] + values[0].toUpperCase();
}

const result = tag`hello ${"world"}`;
```

## Key Features

- **String Interpolation**: Embed expressions with `${expression}`
- **Multi-line Support**: Preserves line breaks and whitespace
- **Expression Evaluation**: Can include any valid JavaScript expression
- **Tagged Templates**: Custom processing of template literals
- **Nested Templates**: Can nest template literals

## Common Use Cases

- Dynamic string creation
- HTML/XML generation
- SQL query building (with caution)
- String formatting and localization
- DSL (Domain Specific Language) creation with tagged templates
