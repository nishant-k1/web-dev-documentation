# Iterators and Iterables Overview

Iterators and Iterables are fundamental concepts in JavaScript that enable iteration over data structures. They form the foundation for `for...of` loops, spread operator, and many built-in methods.

---

## What are Iterables?

An **Iterable** is an object that implements the **Iterable protocol**, meaning it has a method that returns an iterator.

### Iterable Protocol

An object is iterable if it implements the `Symbol.iterator` method:

```javascript
const iterable = {
  [Symbol.iterator]() {
    // Returns an iterator
  },
};
```

---

## What are Iterators?

An **Iterator** is an object that implements the **Iterator protocol**, meaning it has a `next()` method that returns `{ value, done }`.

### Iterator Protocol

An iterator must have a `next()` method that returns:

```javascript
{
  value: any,  // Current value (or undefined if done)
  done: boolean // true if iteration is complete
}
```

---

## Built-in Iterables

JavaScript has many built-in iterables:

- **Arrays**: `[1, 2, 3]`
- **Strings**: `"hello"`
- **Maps**: `new Map()`
- **Sets**: `new Set()`
- **NodeLists**: `document.querySelectorAll()`
- **Arguments**: `arguments` object
- **TypedArrays**: `Uint8Array`, etc.

---

## How `for...of` Works

The `for...of` loop uses the Iterable protocol:

```javascript
const arr = [1, 2, 3];

// for...of internally does:
const iterator = arr[Symbol.iterator]();
let result = iterator.next();

while (!result.done) {
  console.log(result.value);
  result = iterator.next();
}
```

---

## Key Concepts

1. **Iterable Protocol**: Object with `Symbol.iterator` method
2. **Iterator Protocol**: Object with `next()` method returning `{ value, done }`
3. **Generator Functions**: Easy way to create iterators
4. **Built-in Iterables**: Arrays, Strings, Maps, Sets, etc.
5. **Custom Iterables**: Create your own iterable objects
