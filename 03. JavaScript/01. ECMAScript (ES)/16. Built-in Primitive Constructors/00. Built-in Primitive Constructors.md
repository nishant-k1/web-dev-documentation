# 🔑 Symbol & BigInt: Primitive Constructors vs Other Constructors

## Key Difference: What Gets Returned

| Folder                                | Constructor                    | What it returns            | Example                            |
| ------------------------------------- | ------------------------------ | -------------------------- | ---------------------------------- |
| **15** (Symbol, BigInt)               | `Symbol()`, `BigInt()`         | **Primitive value**        | `typeof Symbol()` → `"symbol"`     |
| **16** (String, Number, Boolean)      | `new String()`, `new Number()` | **Object** (wrapper)       | `typeof new String()` → `"object"` |
| **17** (Object, Function, Date, etc.) | `new Object()`, `new Date()`   | **Object** (specific type) | `typeof new Date()` → `"object"`   |
| **18** (Array, Set, Map, etc.)        | `new Array()`, `new Set()`     | **Object** (collection)    | `typeof new Array()` → `"object"`  |
| **19** (Math, JSON, Reflect)          | Not called with `new`          | **Static utility objects** | `Math.random()`                    |

## The key difference for Symbol & BigInt

All other (Wrapper, Collection, General)constructors produce OBJECTS.
Only Symbol & BigInt produce PRIMITIVES.

```js
// Wrapper Constructors - return OBJECTS
new String("hello"); // → [String: 'hello'] (object)
typeof new String("hi"); // → "object"

// General Constructors - return OBJECTS
new Date(); // → Date object
new Function(); // → Function object
typeof new Date(); // → "object"

// Symbol & BigInt - return PRIMITIVES
Symbol(); // → Symbol primitive
typeof Symbol(); // → "symbol" (primitive!)

BigInt(42); // → 42n (primitive)
typeof BigInt(42); // → "bigint" (primitive!)
```
