# Primitive Operations Overview

This document provides an overview of operations that apply to **primitive data types** in JavaScript.

---

## 📋 Operations on Primitives

Primitives support the following operations:

### 1. **Initialization**

Giving a variable its first value.

**Examples:**

```js
let num = 42; // Number
let str = "Hello"; // String
let bool = true; // Boolean
let sym = Symbol("id"); // Symbol
let big = 100n; // BigInt
```

📖 **See:** [14. Initializations](../../14.%20Initializations/) for comprehensive initialization details

---

### 2. **Assignment**

Assigning values to variables.

**Examples:**

```js
let x = 10; // Initialization
x = 20; // Re-assignment
```

📖 **See:** [15. Assignments](../../15.%20Assignments/) for comprehensive assignment details

---

### 3. **Operators**

Arithmetic, logical, comparison, and other operators.

**Examples:**

```js
let sum = 5 + 3; // Arithmetic
let isTrue = x > 5; // Comparison
let result = a && b; // Logical
```

📖 **See:** [13. Operators](../../13.%20Operators/) for comprehensive operator details

---

### 4. **Comparing**

Comparing primitive values.

**Characteristics:**

- Primitives are compared by **value**
- Two primitives with the same value are equal
- Uses `===` or `==` operators

**Examples:**

```js
console.log(5 === 5); // true (same value)
console.log("hello" === "hello"); // true (same value)
```

📖 **See:** [14. Comparing - Comparing Primitives](../../16.%20Comparing/02.%20Comparing%20Primitives.md) for detailed comparison methods

---

### 5. **Copying**

Copying primitive values.

**Characteristics:**

- Copying creates a completely independent copy
- Changes to one variable do not affect others
- Primitives are copied by **value**

**Examples:**

```js
let x = 100;
let y = x; // y gets a copy of x
y = 200;
console.log(x); // 100 (x remains unaffected)
```

📖 **See:** [15. Copying - Copying Primitives](../../17.%20Copying/02.%20Copying%20Primitives.md) for detailed copying methods

---

## 🔑 Key Characteristics of Primitive Operations

### **Immutability**

- ✅ Primitives are **immutable**
- ✅ Any operation creates a **new value**
- ✅ Original value cannot be changed

### **Storage**

- ✅ Stored **by value** in stack memory
- ✅ Direct storage in memory
- ✅ Fixed size

### **Comparison**

- ✅ Compared **by value**, not by reference
- ✅ Two primitives with the same value are always equal

### **Copying**

- ✅ Copied **by value**
- ✅ Creates completely independent copies
- ✅ Changes to copy don't affect original

### **Type Checking**

- ✅ Use `typeof` operator to check type
- ✅ Each primitive has a distinct type

---

## ❌ What Primitives Cannot Do

### **Cannot Mutate**

- ❌ Primitives are immutable - you cannot change their value
- ❌ Operations create new values instead
- 📖 **Note:** Mutation applies only to non-primitives - see [16. Mutation](../../16.%20Mutation/) for details

### **Cannot Have Properties**

- ❌ Primitives cannot have properties or methods directly
- ⚠️ However, JavaScript provides temporary autoboxing for some primitives (String, Number, Boolean)

## 🔄 Operations Overview

| Operation          | How It Works                    | Applies To                  |
| ------------------ | ------------------------------- | --------------------------- |
| **Initialization** | First assignment                | ✅ All primitives           |
| **Assignment**     | Re-assigning values             | ✅ All primitives           |
| **Operators**      | Arithmetic, logical, comparison | ✅ All primitives           |
| **Comparing**      | Value comparison                | ✅ All primitives           |
| **Copying**        | Value copy                      | ✅ All primitives           |
| **Mutation**       | ❌ Not applicable               | ❌ Primitives are immutable |

## 💡 Quick Reference

```js
// All these operations work with primitives
let num = 42; // Initialization
num = 50; // Assignment
let result = num + 10; // Operators
let isEqual = num === 42; // Comparing (by value)
let copy = num; // Copying (by value)
copy = 100;
console.log(num); // 50 (original unchanged - independent copy)
```
