# Parameter Passing Overview

Parameter passing describes how arguments are passed to function parameters in JavaScript. Understanding this is crucial for writing correct and predictable code.

---

## What is Parameter Passing?

When you call a function with arguments, JavaScript passes those values to the function's parameters. The behavior differs based on whether you're passing **primitives** or **non-primitives**.

---

## Types of Parameter Passing

1. **Pass By Value** - For primitive types
2. **Pass By Reference** - For non-primitive types (conceptually)
3. **Pass By Value of Reference** - Technical implementation (JavaScript's actual behavior)

---

## Key Concepts

### Primitives: Pass By Value

- A **copy** of the value is passed
- Changes to the parameter **don't affect** the original
- Applies to: `number`, `string`, `boolean`, `undefined`, `null`, `symbol`, `bigint`

### Non-Primitives: Pass By Reference (Conceptually)

- A **reference** to the object is passed
- Changes to the object **do affect** the original
- Reassigning the parameter **doesn't affect** the original variable
- Applies to: `object`, `array`, `function`, `Date`, `RegExp`, etc.

---

## Why It Matters

Understanding parameter passing helps you:

- **Predict behavior** - Know when changes affect originals
- **Avoid bugs** - Prevent unintended mutations
- **Write better code** - Make informed decisions about copying
- **Debug effectively** - Understand why values change (or don't)
