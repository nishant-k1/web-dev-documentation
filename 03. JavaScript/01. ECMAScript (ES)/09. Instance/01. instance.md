# Instance

- An instance is simply an object whose internal prototype ([[Prototype]]) points to a constructor’s .prototype.
- So:
  - Object → generic term
  - Instance → object created via constructor with proper prototype linkage
- An object created via a constructor function or class, linked to its prototype.

When you do:

```js
class User {
  constructor(name) {
    this.name = name;
  }
}

const u1 = new User("Nishant");
```

**JavaScript:**

- Creates a new object
- Links it to Person.prototype
- Returns it

**That returned object is called:**

- An instance of Person

**🔎 Why Is It Called an “Instance”?**

Because of this linkage:

```js
p1.__proto__ === Person.prototype; // true
p1 instanceof Person; // true
```

**So technically:**

- It is just a normal object.
- But since its internal [[Prototype]] points to Person.prototype, we call it an instance of Person.

**Very Precise Definition**

- An instance is simply an object whose internal prototype ([[Prototype]]) points to a constructor’s .prototype.

Here:

- User → class (constructor)
- u1 → instance of User

You can even check it:

```js
u1 instanceof User; // true
```

- Class = blueprint
- Instance = actual object created from blueprint

## Important Clarification

JavaScript doesn’t use classical class-based OOP internally — it uses prototypes. But the terminology of “instance” is still valid.

When you do: `const arr = new Array();`

- arr is an instance of Array
- Array.prototype is in its prototype chain

## Technically Speaking

**When you run:**: `new User()`

JavaScript:

- Creates a new empty object
- Links it to User.prototype
- Calls the constructor with this set to that object
- Returns that object

That returned object is called the instance.

## Every Object Literal is instance of built in function Object object

Example:
`const obj = {};`

This is still technically:
`obj instanceof Object // true`

So it is an instance of Object.

Because internally:
`{}  → new Object()`

## What instanceof Actually Checks

An object exists somewhere in another object’s prototype chain.

More precisely:

```js
obj instanceof Constructor;
```

This returns true if:

```js
Constructor.prototype;
```

exists in:

```js
obj.__proto__ chain
```

Example

```js
class Person {}
const p = new Person();
console.log(p instanceof Person); // true
```

Because internally:

```js
p.__proto__ === Person.prototype;
```
