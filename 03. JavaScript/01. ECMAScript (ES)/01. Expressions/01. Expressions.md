# What is an Expression?

An **expression produces a value** and can appear anywhere a value is expected.

## Key Characteristics

- **Produces a value** - Every expression evaluates to a value
- **Can be used as values** - Expressions can be used wherever a value is expected
- **Can be nested** - Expressions can contain other expressions
- **Evaluated at runtime** - Expressions are computed when the code runs

## Examples

```js
3 + 4; // Arithmetic expression → 7
"x" + "y"; // String concatenation → "xy"
true ? "yes" : "no"; // Conditional expression → "yes"
myFunction(); // Function call expression → return value
```

Each of these returns something.

## Expression Evaluation Order

Expressions follow operator precedence and associativity rules:

```js
// Precedence example
2 + 3 * 4; // 14 (multiplication before addition)
(2 + 3) * 4; // 20 (parentheses override precedence)

// Associativity example
a = b = c = 5; // Right-to-left (all set to 5)
2 ** (3 ** 2); // Right-to-left (2 ** (3 ** 2))
```

## Expressions vs Statements

| Aspect             | Expression                             | Statement                                       |
| ------------------ | -------------------------------------- | ----------------------------------------------- |
| **Purpose**        | Produces a value                       | Performs an action                              |
| **Returns value**  | Yes                                    | No (or not accessible)                          |
| **Can be used as** | Part of larger expression or statement | Standalone instruction                          |
| **Examples**       | `x + 5`, `myFunction()`, `x > 10`      | `if (x) {}`, `let x = 5`                        |
| **Can be nested**  | Yes (expressions contain expressions)  | Yes (statements contain statements/expressions) |

## Important Notes

1. **Expression Statements**: Any expression followed by a semicolon becomes a statement
2. **Side Effects**: Some expressions have side effects (assignment, function calls, increment/decrement)
3. **Short-Circuit Evaluation**: Logical operators (`&&`, `||`) may not evaluate all operands
4. **Type Coercion**: JavaScript automatically converts types in many expressions
5. **Comma Operator**: The comma operator evaluates left-to-right and returns the last value
