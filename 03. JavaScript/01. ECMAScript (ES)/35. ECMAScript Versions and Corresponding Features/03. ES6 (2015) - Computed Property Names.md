# ES6 (2015) - Computed Property Names

Computed Property Names allow you to use expressions (like variables or function calls) as property keys in object literals.

## What Changed in ES6?

### Before ES6:

```javascript
// Had to define object first, then assign key
const key = "name";
const obj = {};
obj[key] = "John";
```

### After ES6:

```javascript
// Can use expressions directly as keys
const key = "name";
const obj = {
  [key]: "John",
};
```

---

## Syntax

```javascript
const obj = {
  [expression]: value,
};

// Examples
const key = "name";
const obj = {
  [key]: "Nishant",
  [`${key}UpperCase`]: "NISHANT",
  [Math.random()]: "random key",
};
```

---

## Use Cases

- Dynamically generate object keys (e.g., Redux actions)
- Define properties based on function parameters
- Meta programming (dynamic method names)
- In class definitions for computed method names
