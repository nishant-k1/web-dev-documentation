# Object Literal Enhancements Overview

Object literal enhancements are ES6 syntax features that make creating and working with objects more concise and expressive.

---

## What are Object Literal Enhancements?

ES6 introduced three major enhancements to object literal syntax:

1. **Property Shorthand** - Allows omitting property value when key matches variable name
2. **Method Shorthand** - Allows defining methods without the `function` keyword
3. **Computed Property Names** - Allows using expressions as property keys

---

## The Three Enhancements

### 1. Property Shorthand

```javascript
// ❌ Before ES6
const name = "John";
const user = { name: name };

// ✅ ES6
const name = "John";
const user = { name }; // Same as { name: name }
```

See: [02. Property Shorthand.md](./02.%20Property%20Shorthand.md)

---

### 2. Method Shorthand

```javascript
// ❌ Before ES6
const obj = {
  greet: function () {
    return "Hello";
  },
};

// ✅ ES6
const obj = {
  greet() {
    return "Hello";
  },
};
```

See: [03. Method Shorthand.md](./03.%20Method%20Shorthand.md)

---

### 3. Computed Property Names

```javascript
// ❌ Before ES6
const key = "name";
const obj = {};
obj[key] = "John";

// ✅ ES6
const key = "name";
const obj = {
  [key]: "John",
};
```

See: [04. Computed Property Names.md](./04.%20Computed%20Property%20Names.md)

---

## Using All Three Together

You can combine all three enhancements in a single object:

```javascript
const prefix = "user";
const status = "active";

const user = {
  // Property shorthand
  status,

  // Computed property name
  [`${prefix}Id`]: 123,

  // Method shorthand
  greet() {
    return `Hello!`;
  },

  // Computed method name
  [`get${prefix}Info`]() {
    return { id: this.userId };
  },
};
```

## Why Use Object Literal Enhancements?

1. **Cleaner Code** - Less boilerplate, more readable
2. **Modern Syntax** - Standard in ES6+ codebases
3. **Common Patterns** - Used extensively in React, Redux, and modern frameworks
4. **Interview Topics** - Frequently asked in JavaScript interviews
