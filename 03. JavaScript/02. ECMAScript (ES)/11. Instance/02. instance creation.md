# Instance Creation

## Normally instances are created from constructors only, and in most use cases those would be built-in constructors?

This is partly correct, but not fully accurate in modern JavaScript.

✅ 1️⃣ Are instances normally created from constructors?

Yes — most objects ultimately come from constructors.
Even this:

```js
const obj = {};
```

is internally equivalent to:

```js
new Object();
```

So yes — objects are created via constructors under the hood.

✅ 2️⃣ Are most real-world instances created from built-in constructors?

In day-to-day JavaScript apps (especially frontend):
You commonly create:

```js
{
}
[];
new Date();
new Map();
new Set();
new Error();
```

These are built-in constructors.
So yes — built-ins are heavily used.

## ❗ But In Modern Apps (Very Important)

In modern development:

We don’t usually write manual constructor functions.
We mostly use:

- Object literals
- Array literals
- Classes
- Factory functions

Example:

```js
class User {}
const u = new User();
```

Or:

```js
const user = {
  name: "Nishant",
};
```

So while constructors exist, we don’t always think in terms of “constructor functions” anymore.

Reality Check in Production Apps

In real-world frontend apps:
Most objects are:

- Plain objects ({})
- Arrays ([])
- Class instances (models, services, errors)
- Built-in types (Date, Map, Set, Promise)

Rarely:

- Manual function constructors like function Person() {}

## Important Clarification

```js
const obj = {};
```

Its chain is:

```text
obj
  ↓
Object.prototype
  ↓
null
```

So it still comes from the built-in Object constructor.
