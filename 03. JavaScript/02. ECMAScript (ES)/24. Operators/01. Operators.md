# JavaScript Operators (Unary, Binary, Ternary)

Operators are symbols that perform operations on values (operands). JavaScript operators are classified by the number of operands they take.

| Type    | Arity      | Examples                    |
| ------- | ---------- | --------------------------- |
| Unary   | 1 operand  | `!x`, `typeof x`, `++x`     |
| Binary  | 2 operands | `a + b`, `a && b`, `a ?? b` |
| Ternary | 3 parts    | `cond ? x : y`              |

> **Note:** Spread (`...x`) and Rest (`...x`) are **syntax**, not operators.

---

## 🔹 1. Unary Operators

Operate on **one operand**.

### Complete List

| Operator | Example          | Meaning                    |
| -------- | ---------------- | -------------------------- |
| `!`      | `!x`             | Logical NOT                |
| `typeof` | `typeof x`       | Type inspection            |
| `void`   | `void x`         | Always returns `undefined` |
| `delete` | `delete obj.key` | Removes property           |
| `+`      | `+x`             | Converts to number         |
| `-`      | `-x`             | Negates number             |
| `~`      | `~x`             | Bitwise NOT                |
| `++`     | `++i` / `i++`    | Increment                  |
| `--`     | `--i` / `i--`    | Decrement                  |

### Prefix vs Postfix

Unary operators may appear **prefix** (before operand) or **postfix** (after operand):

```js
let i = 5;

// Prefix: increments/decrements first, then uses value
++i; // i becomes 6, returns 6
--i; // i becomes 5, returns 5

// Postfix: uses value first, then increments/decrements
i++; // returns 5, then i becomes 6
i--; // returns 6, then i becomes 5
```

**Note:** `++` and `--` are the only operators that can be both prefix and postfix.

## 🔹 2. Binary Operators

Operate on **two operands**.

### 1. ✔ Arithmetic Operators

- `+` Addition (also string concatenation)
- `-` Subtraction
- `*` Multiplication
- `/` Division
- `%` Modulus (Remainder)
- `**` Exponentiation

**Precedence Order:**

1. `**` (right-to-left associativity)
2. `*`, `/`, `%` (left-to-right)
3. `+`, `-` (left-to-right)

> ⚠️ Note: `++` and `--` are **unary**, not binary.

---

### 2. ✔ Assignment Operators

- `=` Assignment
- `+=` Addition assignment
- `-=` Subtraction assignment
- `*=` Multiplication assignment
- `/=` Division assignment
- `%=` Modulus assignment
- `**=` Exponentiation assignment

**Behavior:**

- `+=` concatenates if either operand is a string
- Others convert operands to numbers

**Modern Assignment Operators:**

- `&&=` Logical AND assignment
- `||=` Logical OR assignment
- `??=` Nullish coalescing assignment

---

### 3. ✔ Comparison Operators

Compare values and return `true` or `false`.

**Equality Operators:**

- `==` Loose equality (performs coercion)
- `!=` Loose inequality
- `===` Strict equality (no coercion)
- `!==` Strict inequality

**Relational Operators:**

- `>` Greater than
- `<` Less than
- `>=` Greater than or equal
- `<=` Less than or equal

**Behavior:**

- `==` performs type coercion before comparison
- `===` does not perform coercion (strict comparison)
- All comparison operators have the same precedence and are left-to-right associative

---

### 4. ✔ Logical Operators

- Short-circuit evaluation means operands are evaluated left to right, and evaluation stops early if the result can be determined without evaluating the remaining operands.
- All of them use short-circuit evaluation. These operators evaluate left to right and short-circuit, meaning the right side may not execute.
  - false && something() // something() never runs
  - true || something() // something() never runs
- Logical operators (`&&,` `||,` `??`) are the core short-circuit operators, but JavaScript also uses the same idea in ternary operator (`?:`) and optional chain (`?.`).

- Evaluates operands from left to right with short-circuiting, meaning the left operand is evaluated first and the right operand is evaluated only if needed.
- If there is chain of operators then for operands left to right evaluation rule is still followed but for the operator precedence rule is followed which are:
  - `!` has higher precedence than all logical operators (`&&,` `||,` `??`)
  - `&&` operators has higher precedence over the `||` operator
  - `??` cannot be mixed/chained with `&&` or `||` without parentheses. But There is no restriction with `!` Eg: `console.log(null ?? false || "fallback");` will return `SyntaxError`
- What does it evaluates to? It doesn't necessarily evaluates to a boolean, rather returns one of the operands. (But inside control flow statements after the complete evaluation of the condition the value gets coerced to boolean. Remember the the evaluated expression value gets coerced not the operands)

- Precedence Order (relevant ones)
  - From highest → lowest: `!` > `&&` > `||` > `??`
  - `!` > `&&`
  - `!` > `||`
  - `!` > `??`
  - `??`, can't be chained with `&&` or `||` without parentheses. But There is no restriction with `!`. `??` is lowest among these But also has a syntax restriction with `&&` / `||`

- Logical Operators can be binary but can be chained, `eg: a && b && c`, but that is interpreted as `(a && b) && c`, so each `&&` still works on 2 operands but expressions can grow longer via chaining.
- So:
  - `&&, ||, ??` are binary operators, each operating on two operands and supporting chaining through left-to-right evaluation.
  - `!` is a unary operator, operating on a single operand and returning a boolean negation.

- if the expression using logical operators is being used as a conditional (inside control statements like or loop conditionals) rather than direct statement in that case then think in terms of truthy/falsy result, not the actual returned value. Because for the condition inside control flow statements
  - 👉 The value of the evaluated expression is coerced to boolean
  - 👉 So the actual returned value of the condition expression doesn’t matter-only whether it’s truthy/falsy
- `??` / `||` → used for default/fallback values
- `!` → used for boolean negation

#### 1. `&&` — Logical AND

- **Binary operator**
- What does it evaluates to? It doesn't necessarily evaluates to a boolean, rather returns one of the operands - Returns the first falsy value, or the last value if all are truthy
- That is:
  - If the left operand is truthy and the right is falsy, it returns the right operand
  - If the left operand is truthy and the right is truthy, it returns the right operand
  - If the left operand is falsy and the right is truthy, it returns the left operand
  - If the left operand is falsy and the right is falsy, it returns the left operand
- **For the condition inside control flow statements**
  - Instead of thinking: “what value does it return?”
    - Think: “If all operands all truthy”
    - Practical mental shortcut: && → all operands must be truthy

```js
true && "JS"; // → 'JS'
false && "JS"; // → false
0 && "hello"; // → 0
"hi" && 123; // → 123
```

#### 2. `||` — Logical OR

- **Binary operator**
- What does it evaluates to? It doesn't necessarily evaluates to a boolean, rather returns one of the operands - Returns the first truthy value, or the last value if all are falsy
- That is:
  - If the left operand is truthy and the right is falsy, it returns the left operand
  - If the left operand is truthy and the right is truthy, it returns the left operand
  - If the left operand is falsy and the right is truthy, it returns the right operand
  - If the left operand is falsy and the right is falsy, it returns the right operand
- **For the condition(A condition = something that will be coerced to boolean) inside control flow statements**
  - Instead of thinking: “what value does it return?”
    - Think: “If anyone operand is truthy”
    - Practical mental shortcut: || → at least one operand must be truthy

```js
false || "default"; // → 'default'
0 || null || "JS"; // → 'JS'
"hi" || "hello"; // → 'hi'
```

#### 3. Nullish Coalescing (`??`)

- **Binary operator**
- `??`, can't be chained with `&&` or `||` without parentheses. But There is no restriction with `!`. Eg: `console.log(null ?? false || "fallback");` will return `SyntaxError`
- Returns the right operand only if the left is null or undefined; otherwise returns the left
- Thats is:
  - if the left is `null` or `undefined`, it returns the right operand only; otherwise returns the left operand
- **For the condition(A condition = something that will be coerced to boolean) inside control flow statements**
  - First resolve value using nullish rule
  - Then check truthiness of that result

#### 4. `!` — Logical NOT

- **Unary operator** (not binary)
- What does it evaluates to? It does necessarily evaluates to a boolean - Negates the truthiness of a value
- returns a boolean value, Negates the value of the expression (converts to boolean and flips it)

```js
!true; // → false
!0; // → true
!!"hi"; // → true (double negation converts to boolean)
```

**Key Points:**

- Does **not** consider `0`, `false`, `''` as nullish
- `!` has higher precedence than all logical operators (`&&,` `||,` `??`)
- `!` → used for boolean negation

```js
null ?? "default"; // → 'default'
undefined ?? "fallback"; // → 'fallback'
0 ?? 42; // → 0 (because 0 is not nullish)
false ?? "yes"; // → false
"" ?? "default"; // → "" (empty string is not nullish)
```

#### ⚠️ Common Pitfalls for logical operators

```js
### ⚠️ Common Pitfalls

0 || "fallback"   // "fallback"
0 ?? "fallback"   // 0

!a ?? b           // b is never used

a ?? b && c       // SyntaxError
```

### 5. Bitwise Operators

Perform operations on binary representations of numbers.

**Bitwise Logical:**

- `&` Bitwise AND
- `|` Bitwise OR
- `^` Bitwise XOR

**Bitwise Shift:**

- `<<` Left shift
- `>>` Right shift (sign-propagating)
- `>>>` Right shift (zero-fill)

> ⚠️ `~` (Bitwise NOT) is **unary**, not binary.

**Note:** Bitwise operators convert operands to 32-bit integers before operation.

---

### 6. String Operator

- `+` String concatenation (when used with strings)

**Behavior:**

- If either operand is a string, `+` performs concatenation
- Otherwise, performs numeric addition

```js
"Hello" + 5; // → "Hello5"
"5" + 3; // → "53"
5 + 3; // → 8
```

## 🔹 3. Ternary (Conditional) Operator

The only ternary operator in JavaScript. Shorthand for `if/else`.

**Syntax:**

```js
condition ? valueIfTrue : valueIfFalse;
```

**Behavior:**

- Evaluates the condition (converts to boolean)
- Returns `valueIfTrue` if condition is truthy
- Returns `valueIfFalse` if condition is falsy
- Only the selected expression is evaluated (short-circuiting)

**Examples:**

```js
let age = 20;
let status = age >= 18 ? "adult" : "minor"; // → "adult"

let x = 5;
let result = x > 0 ? x * 2 : 0; // → 10
```

**Nested Ternary:**

```js
let score = 85;
let grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : "F";
// → "B"
```

> ⚠️ **Note:** Nested ternaries can be hard to read. Consider using `if/else` for complex conditions.
