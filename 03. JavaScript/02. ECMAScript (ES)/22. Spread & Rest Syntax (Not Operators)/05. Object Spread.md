# Object Spread

enumerable properties: properties visible during iteration
Copies only properties that are visible during iteration (enumerable properties).

Spread syntax (`...`) copies enumerable properties from one or more objects into a new object.
Spread syntax in Object allows you to “spread out” the properties of an object into another object.
Here, ...obj1 copies all key-value pairs into obj2.

Only enumerable own properties are copied (not inherited ones).

```js
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1 };

console.log(obj2); // { a: 1, b: 2 }
```

## 1. Shallow Copying Objects

When we spread an object into an empty object, shallow copy gets created.

1. **Mutating top level non-primitive properties, doesn't mutate the original top level non-primitive properties as a new reference is assigned to the top level non-primitive properties**

   ```javascript
   let original = { name: "John", age: 30 };
   let copy = { ...original };
   copy.age = 31;
   // original: { name: "John", age: 30 }
   // Shallow copy: { name: "John", age: 31 }
   ```

2. **Mutating nested level properties, does mutate the original nested level, as top level non primitive properties for both has same reference**

   ```js
   const original = { user: { name: "John" } };
   const copy = { ...original };

   copy.user.name = "Mike";

   console.log(original.user.name); // "Mike" ❗
   ```

3. **Without spread syntax to create shallow copy we had do**

   ```js
   const obj1 = { a: 1, b: 2 };
   const obj2 = Object.assign({}, obj1);

   console.log(obj2); // { a: 1, b: 2 }
   ```

   - How it works:
     {} → target object (new empty object)
     obj1 → source object
     Copies all enumerable properties into the new object

   - Spread is cleaner syntax
   - Both do shallow copy

## 2. Immutably Overriding/Adding Properties to Object (New object is created)

- Order always matters in spread for both Overriding and Adding, It affects:
  - insertion order (used in iteration)
  - which values win (override)

- Object spread copies properties left to right.
- If a key exists → it is overwritten
- If a key does not exist → it is added

```javascript
// Creates new object, doesn't mutate original
let user = { name: "John", age: 30 };
let updated = { ...user, age: 31 };
```

1. **Overriding properties (replacing existing properties)**
   - If key exists, it simply gets replaced
   <!-- - Overriding = when same key exists, later value replaces earlier
   - Order Matters in Object Spread → affects which value wins (override)
   - If the key exists, it simply gets replaced by the one one the right. -->

   ```js
   let obj1 = { a: 1, b: 2 };
   let obj2 = { b: 3, c: 4 };

   // Later properties override earlier ones
   { ...obj1, ...obj2 }  // { a: 1, b: 3, c: 4 }
   { ...obj2, ...obj1 }  // { b: 2, c: 4, a: 1 }
   ```

2. **Adding new properties**
   - If the key doesn’t exist, it simply gets added.

   ```javascript
   // Adding new properties:
   let person = { name: "John", age: 30 };
   let withEmail = { ...person, email: "john@example.com" };
   // Result: { name: "John", age: 30, email: "john@example.com" }
   ```

## 3. Merging objects

- Multiple objects can be merged to become a new single object

  ```js
  const obj1 = { a: 1 };
  const obj2 = { b: 2 };

  const merged = { ...obj1, ...obj2 };

  console.log(merged); // { a: 1, b: 2 }
  ```

## Common Patterns and Use Cases of spread syntax in object

### Pattern 1: Nested spread (very common)

```js
const user = {
  name: "John",
  address: { city: "NYC" },
};

const updated = {
  ...user,
  address: {
    ...user.address,
    city: "LA",
  },
};
```

### Pattern 2: Conditional properties (very common)

```js
const isAdmin = true;

const user = {
  name: "John",
  ...(isAdmin && { role: "admin" }),
};
```

### Pattern 3: Computed keys with spread

```js
const key = "age";

const user = {
  name: "John",
  [key]: 30,
};
```

### Pattern 4: Default Values with Overrides

```js
function createConfig(userConfig) {
  const defaults = {
    theme: "light",
    fontSize: 14,
    language: "en",
  };

  return { ...defaults, ...userConfig };
}

createConfig({ fontSize: 16 });
// Result: { theme: "light", fontSize: 16, language: "en" }
```
