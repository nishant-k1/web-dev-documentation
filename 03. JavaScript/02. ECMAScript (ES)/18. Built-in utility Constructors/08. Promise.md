# Promise Object Constructor

`Promise` is a built-in object for handling asynchronous operations. **Most frequently asked in interviews!**

## What is a Promise?

A Promise is an object that represents the eventual completion or failure of an asynchronous operation. It has three states:

- **Pending**: Initial state, neither fulfilled nor rejected
- **Fulfilled**: Operation completed successfully and has a promise value. Fulfilled doesn't mean raw value.
- **Rejected**: Operation failed and has a promise reason (error) value. Reject doesn't mean raw error value.

Both **Fulfilled** and **Rejected** states are again promise values which can be unpacked using .then or await, while Pending not value at all.

## resolved/resolve

- resolve doesn’t return a raw value
- it always returns a promise holds a value and this value can be raw value or another promise value.
- raw value can only be accessed by either .then or await

`Resolved ≠ Fulfilled (always)`, The promise is locked in to follow another promise OR a value

The return value of resolve is always a promise which can be of two types

1: **Resolved with value → fulfilled**

`resolve returns a value which is a promise that contains that value`

```js
Promise.resolve(5);
// resolved AND fulfilled
```

2: **Resolved with another promise**

`resolve returns a value which is a promise that contains another promise`

```js
let p = Promise.resolve(
  new Promise((resolve) => setTimeout(() => resolve(10), 1000)),
);
```

## Succeeded and Failed

**Succeeded**:This is informal language, “Succeeded” = Fulfilled
**Failed**: This is informal language, “Failed” = Rejected

## Fulfilled vs Settled vs Resolved vs Rejected vs failed vs Succeeded

**Fulfilled/“Succeeded”**: Operation completed successfully and has a value.
**Settled**: Not pending anymore, either Fulfilled or Rejected
**Resolved**: Resolved ≠ Fulfilled (always), The promise is locked in to follow another promise OR a value
**Rejected/Failed**: Operation failed and has a reason (error)

| Term      | Official?           | Meaning                              |
| --------- | ------------------- | ------------------------------------ |
| Pending   | ✅ Yes              | Not completed yet                    |
| Fulfilled | ✅ Yes              | Completed successfully               |
| Rejected  | ✅ Yes              | Completed with error                 |
| Settled   | ✅ Yes              | Either fulfilled or rejected         |
| Resolved  | ⚠️ Yes (but tricky) | Locked to a value or another promise |
| Succeeded | ❌ No               | Same as fulfilled (informal)         |

## Is a resolved promise always fulfilled/successful?

No, resolved promise can return another promise not necessarily always a value and this another promise might get rejected.

## Creating Promises

### 1. Using Constructor

```js
const promise = new Promise((resolve, reject) => {
  // Async operation
  if (success) {
    resolve(value); // might return another promise or a value
  } else {
    reject(error); // Rejected
  }
});
```

### 2. Using Static Method Promise.resolve()

Creates a promise that is:

- already resolved (not fulfilled)
- usually fulfilled (unless value is a promise)

```js
Promise.resolve(2);
```

### 3. Using Static Method Promise.reject()

Creates a promise that is: already rejected

```js
Promise.reject("error");
```

### 4. Using async functions

This automatically returns a promise

```js
async function getData() {
  return 10;
}
```

Above example code is Equivalent to:

```js
function getData() {
  return Promise.resolve(10);
}
```

**There is only one way to create a promise manually — using the Promise constructor.**

We commonly create promises in 4 ways:

- new Promise() (manual creation)
- Promise.resolve()
- Promise.reject()
- async functions (implicit promises)

But Only this gives you full control:

- new Promise((resolve, reject) => {})

Everything else is just:

- wrapping
- shortcut
- or abstraction

## What is returned from the created promise?

- new Promise() (resolve => resolve(x)) → returns a Promise
- Promise.resolve() → returns a Promise
- Promise.reject() → returns a Promise
- async functions (implicit promises) → returns a Promise

✅ Always a Promise

👉 Never returns a raw value

## How to access the actual value from the promise value

Both fulfilled and rejected cases always return a promise object; the value or error is accessed using then, catch, or await.

## Mental model

1. Promise → box
2. Fulfilled → box has value
3. Rejected → box has error
4. .then() / .catch() / await → open the box

## Promise Static Methods

1. all()
2. allSettled()
3. any()
4. race()
5. reject()
6. resolve()
7. try()
8. withResolvers()

### `Promise.all(promises)` - Wait for All

Waits for **all promises to resolve**, or rejects if **any promise rejects** (fail-fast).

```js
const p1 = Promise.resolve(1);
const p2 = Promise.resolve(2);
const p3 = Promise.resolve(3);

Promise.all([p1, p2, p3]).then((values) => {
  console.log(values); // [1, 2, 3]
});

// If any rejects, entire Promise.all rejects
Promise.all([p1, Promise.reject("Error"), p3]).catch((error) => {
  console.error(error); // "Error"
});
```

**Use Cases:**

- Fetching multiple API endpoints in parallel
- All operations must succeed
- Fail-fast behavior

---

### `Promise.race(promises)` - First to Settle

Returns promise that resolves or rejects **as soon as the first promise settles**.

```js
const p1 = new Promise((resolve) => setTimeout(() => resolve("First"), 1000));
const p2 = new Promise((resolve) => setTimeout(() => resolve("Second"), 500));

Promise.race([p1, p2]).then((value) => {
  console.log(value); // "Second" (faster)
});
```

**Use Cases:**

- Timeout patterns
- First response wins
- Racing multiple data sources

### `Promise.allSettled(promises)` ⭐ (ES2020)

Waits for **all promises to settle**, **never rejects**.

```js
const p1 = Promise.resolve(1);
const p2 = Promise.reject("Error");
const p3 = Promise.resolve(3);

Promise.allSettled([p1, p2, p3]).then((results) => {
  console.log(results);
  // [
  //   { status: 'fulfilled', value: 1 },
  //   { status: 'rejected', reason: 'Error' },
  //   { status: 'fulfilled', value: 3 }
  // ]
});
```

**Use Cases:**

- Need results from all promises, even if some fail
- Batch operations where partial success is acceptable

### `Promise.any(promises)` ⭐ (ES2021)

Returns promise that resolves when **first promise resolves**, or rejects if **all promises reject**.

```js
const p1 = Promise.reject("Error 1");
const p2 = Promise.resolve("Success");
const p3 = Promise.reject("Error 2");

Promise.any([p1, p2, p3]).then((value) => {
  console.log(value); // "Success" (first to resolve)
});

// If all reject
Promise.any([Promise.reject("Error 1"), Promise.reject("Error 2")]).catch(
  (error) => {
    console.error(error); // AggregateError: All promises were rejected
  },
);
```

**Use Cases:**

- First success wins
- Fallback strategies
- Redundant data sources

### `Promise.resolve(value)`

Creates a resolved promise.

```js
Promise.resolve(42).then((value) => console.log(value)); // 42
```

### `Promise.reject(reason)`

Creates a rejected promise.

```js
Promise.reject("Error").catch((error) => console.error(error)); // "Error"
```

### `Promise.withResolvers()` ⭐ (ES2024)

Returns an object with `{promise, resolve, reject}` to resolve/reject the promise externally.

```js
const { promise, resolve, reject } = Promise.withResolvers();

// Resolve or reject later
setTimeout(() => resolve("Success"), 1000);

promise.then((value) => {
  console.log(value); // "Success" after 1 second
});
```

**Use Cases:**

- When you need to expose resolve/reject to external code
- Event-driven promise resolution
- Converting callback-based APIs to Promises

```js
// Example: Convert callback to Promise with external control
function createPromise() {
  const { promise, resolve, reject } = Promise.withResolvers();

  // Expose resolve/reject to caller
  return { promise, resolve, reject };
}

const { promise, resolve } = createPromise();
// Caller can resolve whenever needed
resolve("Done");
```

## Instance Methods

### `then(onFulfilled, onRejected?)` ⭐⭐⭐⭐⭐

Handles fulfillment or rejection.

```js
promise.then(
  (value) => {
    // Fulfilled handler
  },
  (error) => {
    // Rejected handler (optional)
  },
);
```

### `catch(onRejected)` ⭐⭐⭐⭐⭐

Handles rejection (shorthand for `.then(null, onRejected)`).

```js
promise.catch((error) => {
  // Error handling
});
```

### `finally(onFinally)` ⭐⭐⭐

Runs regardless of fulfillment or rejection.

```js
promise.finally(() => {
  // Cleanup code (always runs)
});
```

## Comparison Table

| Method                 | Behavior         | Rejects When              | Use Case           |
| ---------------------- | ---------------- | ------------------------- | ------------------ |
| **Promise.all**        | Waits for all    | **Any promise rejects**   | All must succeed   |
| **Promise.race**       | First to settle  | **First promise rejects** | First result wins  |
| **Promise.allSettled** | Waits for all    | **Never rejects**         | Need all results   |
| **Promise.any**        | First to resolve | **All promises reject**   | First success wins |

## Common Patterns

### Timeout Pattern

```js
function withTimeout(promise, timeoutMs) {
  const timeout = new Promise((_, reject) =>
    setTimeout(() => reject(new Error("Timeout")), timeoutMs),
  );
  return Promise.race([promise, timeout]);
}
```

### Parallel Execution

```js
// Execute multiple promises in parallel
const [users, posts, comments] = await Promise.all([
  fetchUsers(),
  fetchPosts(),
  fetchComments(),
]);
```

### Sequential Execution

```js
// Execute promises sequentially
const user = await fetchUser();
const posts = await fetchUserPosts(user.id);
const comments = await fetchPostComments(posts[0].id);
```

### Promise Chaining ⭐⭐⭐⭐⭐

Chaining `.then()` methods to handle sequences of async operations.

```js
fetch("/api/user")
  .then((response) => response.json())
  .then((user) => {
    return fetch(`/api/posts/${user.id}`);
  })
  .then((response) => response.json())
  .then((posts) => {
    console.log(posts);
  })
  .catch((error) => {
    console.error("Error in chain:", error);
  });
```

**Key Points:**

- Each `.then()` returns a new Promise
- Return value becomes the input to next `.then()`
- Single `.catch()` handles errors from any step
- Chain continues even if intermediate steps return non-promises

---

**Note:** Promise execution order (microtask queue, event loop) belongs to the **JavaScript Runtime Environment (JRE)**, not ECMAScript. For details on how promises execute in the runtime, see [29. Handling Async Js/](../../29.%20Handling%20Async%20Js/).

## Error Handling

### Unhandled Promise Rejections

When a Promise rejects and there's no `.catch()` handler, it becomes an **unhandled rejection**.

```js
// Browser
window.addEventListener("unhandledrejection", (event) => {
  console.error("Unhandled rejection:", event.reason);
  event.preventDefault(); // Prevent default console error
});

// Node.js
process.on("unhandledRejection", (reason, promise) => {
  console.error("Unhandled rejection:", reason);
});
```

**Best Practice:** Always add `.catch()` to promise chains:

```js
// ✅ Good
promise.then(handleSuccess).catch(handleError); // Always handle errors

// ❌ Bad - unhandled rejection possible
promise.then(handleSuccess);
```

### Error Propagation

Errors in promise chains propagate to the nearest `.catch()`:

```js
Promise.resolve()
  .then(() => {
    throw new Error("Error 1");
  })
  .then(() => {
    // Skipped due to error above
  })
  .catch((error) => {
    console.error(error); // Catches "Error 1"
    return "recovered";
  })
  .then((value) => {
    console.log(value); // "recovered" (chain continues after catch)
  });
```

## Interview Common Questions

### Q: Difference between Promise.all and Promise.race?

**Promise.all:**

- Waits for ALL promises to resolve
- Rejects if ANY promise rejects (fail-fast)
- Returns array of all results

**Promise.race:**

- Returns first promise to settle (resolve OR reject)
- Doesn't wait for others
- Returns value from first settled promise

### Q: When to use Promise.allSettled?

When you need results from all promises regardless of failures. Unlike Promise.all, it doesn't fail-fast.

### Q: What does Promise.race([]) return?

A promise that **stays pending forever** (no promises to race).

### Q: Implement a timeout pattern?

```js
function withTimeout(promise, timeoutMs) {
  const timeout = new Promise((_, reject) =>
    setTimeout(() => reject(new Error("Timeout")), timeoutMs),
  );
  return Promise.race([promise, timeout]);
}
```

### Q: Promise execution order - what logs first?

```js
console.log("1");
Promise.resolve().then(() => console.log("2"));
setTimeout(() => console.log("3"), 0);
console.log("4");
// Output: 1, 4, 2, 3
```

**Note:** Execution order depends on the runtime environment's event loop (JRE concept, not ECMAScript).

### Q: How does Promise chaining work?

Each `.then()` returns a new Promise. The return value becomes the resolved value for the next `.then()`. If you return a Promise, the chain waits for it to resolve.

### Q: How to handle errors in Promise chains?

Use `.catch()` at the end of the chain. Errors propagate down the chain until caught. After `.catch()`, the chain can continue if you return a value.

```js
promise
  .then(step1)
  .then(step2)
  .catch((error) => {
    // Catches errors from step1 or step2
    return defaultValue; // Chain continues
  })
  .then(step3); // Executes after catch if catch returns value
```
