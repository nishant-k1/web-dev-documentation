# prototype chain

When people say “prototype chain”, they mean:
The chain used for property lookup on an object.

The prototype chain:

- Starts at the object
- Follows [[Prototype]] links
- Ends at null
- Can be any length
- Is determined entirely by how objects are created

The prototype chain is not limited to instance → constructor.prototype → Object.prototype → null. It can be arbitrarily long depending on how objects are linked using Object.create or inheritance.

🔹 Example

```js
function Person() {}
const p1 = new Person();
```

When you access:

```js
p1.toString();
```

JavaScript checks:

```text
p1
  ↓
Person.prototype
  ↓
Object.prototype
  ↓
null
```

That upward path is the prototype chain.

So:

```text
✅ Prototype chain = instance → prototype → prototype → … → null
```

❌ What About Constructor → Function.prototype → Object.prototype?
This:

```text
Person
  ↓
Function.prototype
  ↓
Object.prototype
  ↓
null
```

- This is also a prototype chain…
- But it is the prototype chain of the function object Person.
- It is NOT the prototype chain of p1.

## Clean Mental Model

- Prototype chain = chain of objects
- Constructor = just a function
- Constructor.prototype = commonly used shared object
- Sharing happens because of [[Prototype]] linkage, not because of constructor

## 3️⃣ Important Insight

Each constructor has its own .prototype object:

- Object.prototype
- Array.prototype
- Function.prototype
- Promise.prototype
- etc.

But almost all of them eventually inherit from:

- Object.prototype

Except one special case:

```text
Object.prototype
  ↓
null
```

It is the top of the normal object hierarchy.
