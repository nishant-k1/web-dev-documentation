# Constructor.prototype object vs Constructor object

1️⃣ **Array and Promise Themselves**

```js
typeof Array; // "function"
typeof Promise; // "function"
```

So,

- Array is a function object.
- Promise is a function object.
- They are constructors.

So calling them:

“Array constructor”
“Promise constructor”

2️⃣ **What Is an “Array Object”?**

When we say:

```js
const arr = [];
```

arr is:

- An object
- Whose internal [[Prototype]] points to Array.prototype

So it is perfectly correct to call it:

An Array object
or
An instance of Array

Both are correct.

3️⃣ **What About Array.prototype?**

Array.prototype is:

- Also an object
- But it is the shared prototype object for arrays
- It stores shared methods like push, map, filter

## static vs instance properties and methods

### Static

- Properties and Methods on the Function Constructor Object itself → Static
- They are NOT inherited by the instances.

### instance

- Properties and Methods on the Constructor.prototype Object itself → instance
- They are inherited by array instances.

## What if we have created the instance from another instance not constructors function, in that case, also inheritance happens nad first instance is parent for 2nd instance, now the the properties and methods of 1st instance are inherited by 2nd instance, so all properties and methods of the first instance is instance and cna't be static made?

“Static” is a concept that only makes sense when talking about constructors or classes.
Static means: Belongs to the constructor function itself, not to instances.

But here:

- There is no constructor relationship.
- obj1 is just an object.
- So “static vs instance” doesn’t apply.

**Static vs instance only exists in constructor/class patterns.**

Example:

```js
class A {
  static x() {}
  y() {}
}
```

Here:

- x is static (lives on constructor)
- y is instance method (lives on prototype)

But in plain object-to-object inheritance:

```js
Object.create(obj1);
```

There is no static concept at all.
