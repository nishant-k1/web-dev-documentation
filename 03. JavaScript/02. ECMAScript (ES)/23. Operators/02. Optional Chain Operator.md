# Optional Chain Operator (`?.`)

It is not classified as a unary, binary, or ternary operator in the traditional sense.
Instead, it is considered a short-circuiting property access / call operator.

**Syntax:** `obj?.prop` or `func?.()`

**Behavior:**

- Checks if a property or function exists before accessing/invoking
- Returns `undefined` if the left-hand side is `null` or `undefined`
- Prevents `TypeError` from accessing properties of `null`/`undefined`

**Examples:**

```js
let user = {};
let name = user?.name ?? "unknown"; // → "unknown"

let obj = null;
obj?.property; // → undefined (no error)

let func = null;
func?.(); // → undefined (no error)
```

**Chaining:**

```js
let user = { address: { city: "NYC" } };
user?.address?.city; // → "NYC"

let user2 = null;
user2?.address?.city; // → undefined (no error)
```
