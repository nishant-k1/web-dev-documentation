The `console` API is the **most frequently used Web API** by JavaScript developers. It provides methods for logging, debugging, and inspecting code execution. **Used every single day!**

---

## Basic Console Methods

### console.log()

Logs a message to the console.

```js
console.log("Hello, World!");
console.log("User:", { name: "John", age: 30 });
console.log("Count:", 42);
```

### console.error()

Logs an error message (usually displayed in red).

```js
console.error("Something went wrong!");
console.error(new Error("Failed to fetch data"));
```

### console.warn()

Logs a warning message (usually displayed in yellow).

```js
console.warn("This is deprecated!");
console.warn("Memory usage is high");
```

### console.info()

Logs an informational message.

```js
console.info("Application started");
console.info("User logged in successfully");
```

### console.debug()

Logs a debug message (may require verbose logging enabled).

```js
console.debug("Debug information:", data);
```

## Advanced Console Methods

### console.table()

Displays data in a table format. **Great for arrays and objects!**

```js
// Table from array of objects
const users = [
  { name: "John", age: 30, city: "NYC" },
  { name: "Jane", age: 25, city: "LA" },
];

console.table(users);
// Displays as a nice table!

// Table from object
const data = { name: "John", age: 30, city: "NYC" };
console.table(data);
```

### console.dir()

Displays an interactive list of object properties (better for objects than `console.log`).

```js
const obj = { a: 1, b: { c: 2, d: 3 } };
console.dir(obj); // Interactive tree view
```

**console.log vs console.dir:**

```js
const element = document.querySelector("button");

console.log(element); // Shows HTML representation
console.dir(element); // Shows object properties tree
```

### console.group() / console.groupEnd()

Groups related log messages together.

```js
console.group("User Details");
console.log("Name: John");
console.log("Age: 30");
console.log("City: NYC");
console.groupEnd();

// Collapsible group in console
```

### console.groupCollapsed()

Creates a collapsed group (expanded by default).

```js
console.groupCollapsed("API Calls");
console.log("GET /api/users");
console.log("POST /api/login");
console.groupEnd();
```

### console.time() / console.timeEnd()

Measures execution time.

```js
console.time("Array Processing");
// Some operation
for (let i = 0; i < 1000000; i++) {
  // processing
}
console.timeEnd("Array Processing");
// Output: Array Processing: 15.234ms
```

### console.trace()

Prints a stack trace showing where the function was called from.

```js
function a() {
  b();
}

function b() {
  c();
}

function c() {
  console.trace("Trace here");
}

a();
// Shows call stack: c -> b -> a
```

### console.assert()

Logs an error message if assertion is false.

```js
const age = 15;
console.assert(age >= 18, "User must be 18 or older");
// Logs error if age < 18

// Useful for debugging
console.assert(arr.length > 0, "Array should not be empty");
```

### console.count() / console.countReset()

Counts how many times a label is called.

```js
console.count("Button clicks"); // 1
console.count("Button clicks"); // 2
console.count("Button clicks"); // 3
console.countReset("Button clicks");
console.count("Button clicks"); // 1
```

### console.clear()

Clears the console.

```js
console.clear(); // Clears all console output
```

## String Substitution

Console methods support placeholders for string formatting.

```js
const name = "John";
const age = 30;

console.log("Name: %s, Age: %d", name, age);
// Output: Name: John, Age: 30

// Placeholders:
// %s - String
// %d or %i - Integer
// %f - Float
// %o - Object
// %c - CSS styling

console.log("%cStyled text", "color: red; font-size: 20px;");
```

### CSS Styling with %c

```js
console.log(
  "%cImportant Message",
  "color: red; font-weight: bold; font-size: 16px;"
);

console.log(
  "%cSuccess! %cDetails",
  "color: green; font-weight: bold;",
  "color: gray;"
);
```

## Common Use Cases

### 1. Debugging

```js
function processData(data) {
  console.log("Input:", data);
  const result = data.map((item) => item * 2);
  console.log("Output:", result);
  return result;
}
```

### 2. Performance Measurement

```js
console.time("API Call");
await fetch("/api/data");
console.timeEnd("API Call");
```

### 3. Inspecting Objects

```js
const complexObject = {
  user: {
    profile: {
      settings: { theme: "dark" },
    },
  },
};

console.dir(complexObject); // Interactive exploration
```

### 4. Table Display

```js
const sales = [
  { product: "Widget", qty: 10, price: 5.0 },
  { product: "Gadget", qty: 5, price: 10.0 },
];

console.table(sales); // Beautiful table view
```

### 5. Grouped Logging

```js
function login(user) {
  console.group("Login Process");
  console.log("Username:", user.name);
  console.log("Timestamp:", new Date());
  console.log("IP Address:", user.ip);
  console.groupEnd();
}
```

## Interview Common Questions

### Q: What's the difference between console.log and console.dir?

**console.log:**

- Shows the object's string representation
- For DOM elements: shows HTML
- Better for quick inspection

**console.dir:**

- Shows interactive property tree
- For DOM elements: shows object properties
- Better for exploring object structure

```js
const element = document.querySelector("button");
console.log(element); // <button>Click</button>
console.dir(element); // Interactive object tree
```

### Q: What are some useful console methods for debugging?

1. **console.log()** - Basic logging
2. **console.table()** - Display data in tables
3. **console.time() / timeEnd()** - Measure performance
4. **console.trace()** - Stack trace
5. **console.group()** - Organize logs
6. **console.assert()** - Conditional logging

### Q: How to style console output?

Use `%c` placeholder with CSS:

```js
console.log("%cRed text", "color: red; font-weight: bold;");
```

### Q: How to measure execution time?

```js
console.time("label");
// Your code
console.timeEnd("label");
```

### Q: What does console.table() do?

Displays array of objects or object properties in a tabular format. Very useful for debugging data structures.

```js
console.table([{ name: "John", age: 30 }]);
// Shows as table
```

## Best Practices

### ‚úÖ Do:

- Use `console.table()` for arrays/objects
- Use `console.time()` for performance debugging
- Use `console.group()` to organize related logs
- Use `console.dir()` to inspect object properties
- Use `console.assert()` for debugging conditions

### ‚ùå Don't:

- Leave console.log in production code (use build tools to remove)
- Overuse console.log (can slow down performance)
- Use console.log for error handling (use proper error handling)

### Production Tip

Remove console statements in production:

```js
// Using a build tool or manually
if (process.env.NODE_ENV !== "production") {
  console.log("Debug info");
}
```

Or use a utility:

```js
const debug = {
  log: (...args) => {
    if (process.env.NODE_ENV === "development") {
      console.log(...args);
    }
  },
};
```

## Browser Compatibility

All modern browsers support console API methods. However:

- Some methods may have slight variations
- `console.table()` might not be available in very old browsers
- Always check browser DevTools for available methods
