# FAQs BOM

## BOM vs window

- BOM ‚â† window
- But window is the core object of the BOM.
- BOM (Browser Object Model) is the collection of browser-specific objects that allow JavaScript to interact with the browser outside of the web page's content.
- window is the root object that holds all BOM-related properties and methods.
- window is just the entry point to access BOM features.

| Concept      | Is `window`?             | Explanation                                                                  |
| ------------ | ------------------------ | ---------------------------------------------------------------------------- |
| **BOM**      | ‚ùå Not equal to `window` | It's a group of browser-specific objects (and `window` is just one of them). |
| **`window`** | ‚úÖ Part of BOM           | Root/global object that contains the other BOM APIs.                         |

## DOM vs document

- DOM ‚â† document
- But document is the main entry point into the DOM.
- DOM (Document Object Model) is a tree-like structure representing the HTML (or XML) content of a web page.
- It lets JavaScript read, traverse, and manipulate the page structure dynamically.
- document is a global object provided by the browser to let you interact with the DOM.
- It's your main access point to the DOM.

| Concept        | Is it the same as...? | Explanation                                             |
| -------------- | --------------------- | ------------------------------------------------------- |
| **DOM**        | ‚ùå `document`         | The actual **model/structure** of your HTML in memory.  |
| **`document`** | ‚úÖ Entry to DOM       | JavaScript **object** that gives you access to the DOM. |

Analogy
Think of the DOM as the entire house (HTML structure),
and document as the door you use to enter and modify the house.

**What is window?**

`Being a property of window does not define the API group it belongs to.`

## webAPIs and BOM is different? setTimeout is BOM not a webAPI while fetch is webAPI?

Web APIs = Big umbrella term
Web APIs are all browser-provided APIs available to JavaScript that are not part of the core JS language (like Array, Promise, etc.).

They include:
DOM API ‚Äì Working with HTML (document, Element, etc.)
BOM API ‚Äì Interacting with the browser (location, alert, screen, etc.)
Modern APIs ‚Äì Like fetch, WebSocket, Notification, etc.

BOM = A smaller part of Web APIs
BOM is one of the oldest subsets of the Web API. It deals only with the browser window and environment.
Classic BOM examples:

window

- `alert(), confirm(), prompt()`
- `setTimeout(), setInterval()`
- `location, navigator, screen, history`

---

## So what is setTimeout?

It's historically considered BOM (because it came early, tied to browser window behavior).
But technically, it's also part of Web APIs, because it's provided by the browser, not the JS engine (like V8).

## What about fetch?

fetch() is a modern Web API, introduced later.
It is not part of the BOM, because it's not tied to the browser window or UI ‚Äî it's for networking.
You won't find it on old BOM API lists.

| Feature       | Part of BOM? | Part of Web API? | Explanation                                |

## BOM vs DOM: Where Does `HTMLElement` Belong?

`HTMLElement` is **not** part of the BOM. It is part of the **DOM**.

---

## üìÅ BOM (Browser Object Model)

The BOM provides tools to interact with the **browser environment**, not the page content.

- Includes objects like:
  - `window`
  - `navigator`
  - `location`
  - `history`
  - `screen`
  - `alert()`, `confirm()`, `prompt()`
  - `setTimeout()`, `setInterval()`, etc.
- Purpose: Interact with browser settings, tabs, URLs, and user environment.

---

## üß© DOM (Document Object Model)

The DOM represents the **structure of the HTML document** ‚Äî the elements you see and interact with.

- Includes:
  - `document`
  - `Element`, `Node`, `HTMLElement`
  - `HTMLDivElement`, `HTMLImageElement`, etc.
  - Properties/methods like `.innerHTML`, `.appendChild()`, `.style`, etc.
- Purpose: Access and manipulate the page's content and structure.

---

## ‚úÖ So What is `HTMLElement`?

- `HTMLElement` is a **class (constructor)** that all HTML elements inherit from.
- It is available globally:
  ```js
  console.log(window.HTMLElement === HTMLElement); // true
  ```
- Even though it's a property of `window`, it's **not part of BOM** ‚Äî it's part of the **DOM API**.

- **DOM = Document**: Things _inside_ your HTML (elements, content, tags)
