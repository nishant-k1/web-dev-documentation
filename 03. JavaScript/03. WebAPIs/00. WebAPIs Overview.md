# WebAPIs Overview

## What Are WebAPIs?

**WebAPIs (Web Application Programming Interfaces)** are browser-provided APIs that extend JavaScript's capabilities beyond the ECMAScript language specification. They allow JavaScript to interact with the browser environment, the DOM, network, storage, and more.

---

## ‚ö†Ô∏è Important: Part of JavaScript Runtime Environment (JRE)

**This folder (`03. WebAPIs/`) is part of the JavaScript Runtime Environment (JRE).**

WebAPIs are **not** part of the ECMAScript language specification. Instead, they are provided by the **browser's runtime environment** to enable JavaScript to:

- Interact with the DOM (Document Object Model)
- Handle events (mouse, keyboard, touch, etc.)
- Make network requests (fetch, XMLHttpRequest)
- Store data (localStorage, sessionStorage, IndexedDB)
- Access browser features (window, navigator, location)
- Perform background tasks (Web Workers)
- And much more...

---

## Relationship to ECMAScript vs JRE

| Category       | What It Is             | Examples                                                         |
| -------------- | ---------------------- | ---------------------------------------------------------------- |
| **ECMAScript** | Language specification | `Promise`, `Array`, `Object`, `String`, `async/await`            |
| **JRE**        | Runtime environment    | Event Loop, Call Stack, Heap, Task Queues                        |
| **WebAPIs**    | Browser-provided APIs  | `fetch`, `setTimeout`, `localStorage`, `DOM`, `addEventListener` |

**Key Point:** JavaScript runs in a **runtime environment** (browser or Node.js) that provides WebAPIs beyond the core language.
