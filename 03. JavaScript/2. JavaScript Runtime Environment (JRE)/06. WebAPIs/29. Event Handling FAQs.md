# Event Handling FAQs

1. target vs currentTarget
   `target` = Who initiated the event (the origin of the click)
   `currentTarget` = Who is handling the event (the listener's element)

   | Property        | Description                                 |
   | --------------- | ------------------------------------------- |
   | `target`        | The actual element that triggered the event |
   | `currentTarget` | The element the listener is bound to        |

8. What does event.preventDefault() do?

   1. Stops a link (<a>) from navigating
   2. Stops a form from submitting
   3. Prevents right-click context menu
   4. Stops text selection or drag-drop

   Use preventDefault() when you want to override or control the browser's default behavior.

   | Use Case                | Why Use `preventDefault()`?                                     |
   | ----------------------- | --------------------------------------------------------------- |
   | `<form onSubmit>`       | Prevent page reload to handle with JS (like validation or AJAX) |
   | `<a href="#">`          | Stop navigation (often used for buttons styled as links)        |
   | Drag-and-drop           | Prevent browser from opening dropped files                      |
   | Right-click menu        | Disable context menu on images or UI elements                   |
   | Keyboard shortcuts      | Prevent default browser shortcuts (e.g. `Ctrl+S`)               |
   | `touchmove` or `scroll` | Stop scrolling on mobile while swiping over a specific element  |

   Avoid using preventDefault() when:

   - You're not overriding any default browser behavior.
   - You're just listening to an event for analytics, logging, or side-effects.
   - You're not working with an event that has a default action.

     | Question                         | Answer                                                         |
     | -------------------------------- | -------------------------------------------------------------- |
     | What does `preventDefault()` do? | Prevents the default browser behavior                          |
     | When is it required?             | When you want to stop navigation, form submit, drag-drop, etc. |
     | When is it unnecessary?          | If you're not overriding a default browser action              |
     | Does it stop event propagation?  | ‚ùå No! Use `stopPropagation()` for that                        |

üîπ 2. Links

| Element          | Default Behavior |
| ---------------- | ---------------- |

| `drop`                    | May open a file if dropped |
| `mousedown` + `mousemove` | Starts text selection      |
