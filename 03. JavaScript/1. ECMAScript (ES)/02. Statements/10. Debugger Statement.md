# Debugger Statement

Invokes any available debugging functionality. If no debugger is available, this statement has no effect.

## Syntax

```js
debugger;
```

## Examples

### Basic Usage

```js
function problematicFunction() {
  let x = 10;
  debugger; // execution pauses here if debugger is attached
  return x * 2;
}
```

### In Loops

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    debugger; // Pause when i is 5
  }
  console.log(i);
}
```

### Conditional Debugging

```js
function processData(data) {
  if (data.length === 0) {
    debugger; // Pause if empty data
  }
  // process data
}
```

## How It Works

1. When JavaScript encounters `debugger;`, it checks if a debugger is attached
2. If a debugger is available (browser DevTools, Node.js inspector, etc.), execution pauses
3. If no debugger is available, the statement is ignored and execution continues

## Use Cases

1. **Breakpoint in code** - Set breakpoints directly in source code
2. **Conditional debugging** - Only pause under specific conditions
3. **Development debugging** - Quick way to inspect state without using IDE breakpoints

## Important Notes

- **No effect in production** - If no debugger is attached, statement is ignored
- **Must be removed or guarded** - Should not be left in production code
- **Works with all debuggers** - Browser DevTools, Node.js inspector, VS Code debugger, etc.
- **Better than console.log** - Allows inspection of full execution context

## Best Practices

```js
// ✅ Good - Remove before production
function debugFunction() {
  debugger; // TODO: Remove before production
  // code
}

// ✅ Better - Guard with environment check
function debugFunction() {
  if (process.env.NODE_ENV === "development") {
    debugger;
  }
  // code
}

// ✅ Best - Use conditional compilation or build tools to remove
```

## Browser DevTools

When using browser DevTools:

- Open DevTools before running code
- Execution will pause at `debugger;` statement
- You can inspect variables, call stack, and step through code
- Similar to setting a breakpoint in the Sources panel
