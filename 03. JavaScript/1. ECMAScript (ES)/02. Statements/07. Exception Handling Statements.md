# Exception Handling Statements

Handle errors and exceptions.

**Note:** The `throw` statement is also classified as a Jump Statement since it transfers control, but it's covered here in detail as it's primarily used for exception handling.

## try-catch

Catches and handles errors.

```js
try {
  // code that might throw an error
  riskyOperation();
} catch (error) {
  // handle the error
  console.error(error);
}
```

## try-catch-finally

Catches errors and always executes cleanup code.

```js
try {
  // code
} catch (error) {
  // handle error
} finally {
  // always executes, regardless of success or failure
  cleanup();
}
```

## try-finally

Executes cleanup code without catching errors.

```js
try {
  // code
} finally {
  // always executes
  cleanup();
}
```

## throw

Throws an exception (also classified as a Jump Statement).

```js
// Throw Error object
throw new Error("Error message");

// Throw string
throw "String error";

// Throw any value
throw 42;
throw { code: 500, message: "Server error" };
```

## Examples

### Basic Error Handling

```js
try {
  let result = divide(10, 0);
} catch (error) {
  console.error("Division failed:", error.message);
}
```

### Multiple Catch Blocks (Not in JavaScript)

JavaScript doesn't support multiple catch blocks. Use conditional logic instead:

```js
try {
  riskyOperation();
} catch (error) {
  if (error instanceof TypeError) {
    // handle TypeError
  } else if (error instanceof ReferenceError) {
    // handle ReferenceError
  } else {
    // handle other errors
  }
}
```

### Finally for Cleanup

```js
let file;
try {
  file = openFile("data.txt");
  processFile(file);
} catch (error) {
  console.error("Error processing file:", error);
} finally {
  if (file) {
    closeFile(file); // Always close, even if error occurred
  }
}
```

## Important Notes

- **try** block contains code that might throw an error
- **catch** block handles the error (parameter is optional in ES2019+)
- **finally** block always executes, even if there's a return statement
- **throw** can throw any value, not just Error objects
- Uncaught exceptions propagate up the call stack
