# Import/Export Statements (ES6 Modules)

Module-related statements (though technically declarations, they're also statements).

## Export Statements

### Named Exports

```js
// Export individual declarations
export const name = "John";
export function greet() {}
export class MyClass {}

// Export list
const name = "John";
function greet() {}
export { name, greet };

// Export with renaming
export { name as userName, greet as sayHello };
```

### Default Export

```js
// Default export (one per module)
export default class MyClass {}
export default function myFunction() {}
export default { name: "John", age: 30 };

// Or
const MyClass = class {};
export default MyClass;
```

### Re-exporting

```js
// Re-export from another module
export { name, age } from "./other-module.js";

// Re-export default
export { default } from "./other-module.js";

// Re-export all
export * from "./other-module.js";
```

## Import Statements

### Named Imports

```js
// Import specific named exports
import { name, age } from "./module.js";

// Import with renaming
import { name as userName, age as userAge } from "./module.js";

// Import all as namespace
import * as utils from "./utils.js";
// Usage: utils.functionName()
```

### Default Import

```js
// Import default export
import MyClass from "./module.js";
import myFunction from "./module.js";
```

### Mixed Imports

```js
// Import both default and named
import MyClass, { name, age } from "./module.js";

// Import default with namespace
import defaultExport, * as namedExports from "./module.js";
```

### Side-effect Only Import

```js
// Import module for side effects only (no bindings)
import "./polyfills.js";
import "./styles.css";
```

## Dynamic Import

Dynamic imports use function syntax, not statement syntax:

```js
// Dynamic import (returns Promise)
const module = await import("./module.js");
const { name, age } = await import("./module.js");
```

## Examples

### Module File (module.js)

```js
// Named exports
export const PI = 3.14159;
export function calculateArea(radius) {
  return PI * radius * radius;
}

// Default export
export default class Circle {
  constructor(radius) {
    this.radius = radius;
  }
}
```

### Import File (main.js)

```js
// Import named and default
import Circle, { PI, calculateArea } from "./module.js";

const circle = new Circle(5);
console.log(calculateArea(circle.radius));
```

## Important Notes

- **Module scope** - Imports/exports are hoisted to top of module
- **Strict mode** - All modules are automatically in strict mode
- **Top-level only** - Cannot be used inside blocks, functions, or conditionals
- **File extension** - Include `.js` extension in imports (required in some environments)
- **Static analysis** - Import/export statements are statically analyzable

## Module Features

1. **Encapsulation** - Each module has its own scope
2. **Reusability** - Code can be shared across files
3. **Tree-shaking** - Unused exports can be eliminated by bundlers
4. **Static analysis** - Dependencies can be analyzed at build time
