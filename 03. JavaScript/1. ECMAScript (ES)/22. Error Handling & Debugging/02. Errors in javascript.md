# Errors in JavaScript

In JavaScript, **errors** and **exceptions** are the same thing. Error handling means catching and managing these errors to prevent your application from crashing.

---

## Error Constructor

JavaScript has a native `Error` constructor function that creates error objects.

```javascript
new Error("oopsie");
```

This creates an error object but doesn't throw it. To actually throw an error, use the `throw` keyword.

---

## The `throw` Keyword

The `throw` statement is used to throw (raise) an error. When an error is thrown:

- The current function execution **stops immediately**
- Control is passed up the call stack
- The script continues looking for error handlers (catch blocks)

```javascript
throw "string";
return 4 + 3; // Won't execute - execution stopped at throw
```

**You can throw anything in JavaScript:**

```javascript
throw "string error";
throw 42;
throw { error: "custom object" };
throw new Error("proper error");
```

> ðŸ’¡ **Best Practice**: Always throw `Error` objects (or subclasses) rather than primitives for better debugging.

---

## Error Properties

Error objects have two main properties:

1. **`message`** - The error message
2. **`stack`** - The stack trace showing where the error occurred

```javascript
const myError = new Error("oopsie");

// Error message
myError.message; // "oopsie"

// Stack trace
myError.stack; // Shows the call stack where error occurred
```

---

## Error Propagation

When an error is thrown:

1. **Current execution stops** - The function where the error occurred stops executing
2. **Call stack unwinding** - JavaScript looks up the call stack for an error handler
3. **Handler search** - It checks each execution context for a `catch` block
4. **Runtime handling** - If no handler is found, the runtime handles it:
   - **Browser**: Calls `window.onerror()` (shows red error in console)
   - **Node.js**: Calls `process.on('uncaughtException')` (crashes the process)

```javascript
function level1() {
  level2(); // Error propagates up
}

function level2() {
  level3(); // Error propagates up
}

function level3() {
  throw new Error("Something went wrong"); // Error starts here
}

level1(); // Error bubbles up through all functions
```
