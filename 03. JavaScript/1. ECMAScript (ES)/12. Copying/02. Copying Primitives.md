# Copying Primitives

Primitives in JavaScript are copied by value, creating completely independent copies.

## Copy by Value

- Assigning a primitive value to another variable creates a completely independent copy.
- Changing the copied value does not affect the original value, as primitives are immutable.

```js
let a = 42;
let b = a; // Copy by value
b = 50;
console.log(a); // 42 (independent copy)
console.log(b); // 50
```

## Methods for Copying Primitives

### 1. Using Assignment Operator

```js
let str1 = "Hello";
let str2 = str1; // Independent copy
str2 = "World";
console.log(str1); // "Hello" (unchanged)
console.log(str2); // "World"
```

### 2. Using `String.prototype.slice()` (for Strings)

```js
let str1 = "JavaScript";
let str2 = str1.slice();
console.log(str2); // "JavaScript"
```

### 3. Using Template Literals

```js
let str1 = "Learn";
let str2 = `${str1}`;
console.log(str2); // "Learn"
```

### 4. Using Math Operations (for Numbers)

```js
let num1 = 10;
let num2 = num1 + 0; // Copy by calculation
console.log(num2); // 10
```

### 5. Using Conversions (for Primitives that Support It)

```js
let num1 = 42;
let num2 = Number(num1); // Copy by type conversion
console.log(num2); // 42
```

## Important Notes

- **Primitives are immutable** - You cannot modify a primitive value, only reassign the variable
- **Copy by value is automatic** - Simply assigning creates an independent copy
- **No special methods needed** - Unlike objects, primitives don't require special copying techniques
