# Built-in Error Types

JavaScript provides several built-in error types. Understanding these helps you identify and handle errors more effectively.

---

## Error Types Overview

| Error Type       | When It Occurs              | Example                       |
| ---------------- | --------------------------- | ----------------------------- |
| `Error`          | Generic error (base class)  | `new Error("message")`        |
| `TypeError`      | Wrong type operation        | `null.property`               |
| `ReferenceError` | Undefined variable          | `undefinedVariable`           |
| `SyntaxError`    | Syntax mistake              | `function(` (missing closing) |
| `RangeError`     | Out of range value          | `new Array(-1)`               |
| `URIError`       | URI encoding/decoding error | `decodeURIComponent("%")`     |
| `EvalError`      | eval() error (rare)         | `eval("invalid code")`        |

---

## 1. Error (Base Class)

The base class for all errors. Other error types extend from this.

```javascript
const error = new Error("Something went wrong");
console.log(error.name); // "Error"
console.log(error.message); // "Something went wrong"
```

---

## 2. TypeError

Occurs when a value is not of the expected type.

### Common Causes

```javascript
// Accessing property on null/undefined
null.property; // TypeError: Cannot read property 'property' of null
undefined.method(); // TypeError: Cannot read property 'method' of undefined

// Calling non-function
const obj = {};
obj(); // TypeError: obj is not a function

// Wrong method on type
const num = 123;
num.push(4); // TypeError: num.push is not a function

// Destructuring non-iterable
const [a, b] = null; // TypeError: null is not iterable
```

### Handling TypeError

```javascript
function safeAccess(obj, prop) {
  try {
    return obj[prop];
  } catch (error) {
    if (error instanceof TypeError) {
      console.log("Invalid object or property");
      return null;
    }
    throw error;
  }
}
```

## 3. ReferenceError

Occurs when trying to access a variable that doesn't exist.

### Common Causes

```javascript
// Undefined variable
console.log(undefinedVariable); // ReferenceError: undefinedVariable is not defined

// Before declaration (TDZ)
console.log(x); // ReferenceError: Cannot access 'x' before initialization
let x = 5;

// Wrong scope
function outer() {
  let innerVar = "secret";
}
console.log(innerVar); // ReferenceError: innerVar is not defined
```

### Handling ReferenceError

```javascript
function safeVariableAccess(varName) {
  try {
    return eval(varName); // Not recommended, but example
  } catch (error) {
    if (error instanceof ReferenceError) {
      console.log(`Variable ${varName} is not defined`);
      return undefined;
    }
    throw error;
  }
}
```

---

## 4. SyntaxError

Occurs when JavaScript code has syntax mistakes. Usually caught during parsing, not runtime.

### Common Causes

```javascript
// Missing closing bracket
function test( { // SyntaxError: Unexpected end of input

// Invalid JSON
JSON.parse('{invalid json}'); // SyntaxError: Unexpected token

// Reserved word misuse
const class = "test"; // SyntaxError: Unexpected token 'class'

// Missing quotes
const str = hello; // SyntaxError: hello is not defined (if not declared)
```

### Note

SyntaxError is usually caught **before** code runs (during parsing), so try-catch won't catch it unless it's in `eval()` or `Function()`.

```javascript
try {
  eval("function test("); // SyntaxError caught here
} catch (error) {
  if (error instanceof SyntaxError) {
    console.log("Syntax error in eval");
  }
}
```

## 5. RangeError

Occurs when a value is outside the allowed range.

### Common Causes

```javascript
// Invalid array length
new Array(-1); // RangeError: Invalid array length
new Array(Number.MAX_VALUE); // RangeError: Invalid array length

// Invalid string method parameters
"hello".substring(5, 2); // RangeError (in strict mode)
"hello".repeat(-1); // RangeError: Invalid count value

// Invalid number methods
(123).toFixed(-1); // RangeError: toFixed() digits must be between 0 and 100
```

### Handling RangeError

```javascript
function safeArrayLength(length) {
  try {
    return new Array(length);
  } catch (error) {
    if (error instanceof RangeError) {
      console.log("Invalid array length");
      return [];
    }
    throw error;
  }
}
```

---

## 6. URIError

Occurs when URI encoding/decoding operations fail.

### Common Causes

```javascript
// Invalid URI encoding
decodeURIComponent("%"); // URIError: URI malformed
decodeURI("%E0%A4%A"); // URIError: URI malformed

// Invalid characters in URI
encodeURI("\uD800"); // URIError: URI malformed (surrogate pair)
```

### Handling URIError

```javascript
function safeDecodeURI(uri) {
  try {
    return decodeURIComponent(uri);
  } catch (error) {
    if (error instanceof URIError) {
      console.log("Invalid URI");
      return uri; // Return original if decode fails
    }
    throw error;
  }
}
```

---

## 7. EvalError

Occurs when `eval()` is used incorrectly. **Rarely seen** in modern JavaScript.

```javascript
// EvalError is rarely thrown in modern JavaScript
// It was used in older versions for eval() misuse
// Now, eval() errors usually throw SyntaxError or ReferenceError
```

## Checking Error Types

### Using `instanceof`

```javascript
try {
  riskyOperation();
} catch (error) {
  if (error instanceof TypeError) {
    // Handle TypeError
  } else if (error instanceof ReferenceError) {
    // Handle ReferenceError
  } else if (error instanceof RangeError) {
    // Handle RangeError
  } else {
    // Handle other errors
  }
}
```

### Using `error.name`

```javascript
try {
  riskyOperation();
} catch (error) {
  switch (error.name) {
    case "TypeError":
      // Handle TypeError
      break;
    case "ReferenceError":
      // Handle ReferenceError
      break;
    default:
    // Handle other errors
  }
}
```

## Best Practices

### ✅ Do

```javascript
// Check error types specifically
catch (error) {
  if (error instanceof TypeError) {
    // Handle type errors
  }
}

// Provide helpful error messages
throw new TypeError("Expected number, got " + typeof value);
```

### ❌ Don't

```javascript
// Don't catch all errors the same way
catch (error) {
  // All errors handled the same - not ideal
}

// Don't ignore error types
catch (error) {
  console.log("Error occurred"); // Too generic
}
```
