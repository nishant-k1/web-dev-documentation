## Major ES10 Features

### **Optional Chaining (?.)**

ğŸ“– See: [08. Operators - Optional Chaining](../08.%20Operators/1.%20Operators.md#-4-optional-chaining-)

**Syntax:** `obj?.prop`, `func?.()`, `arr?.[index]`

Allows safe property access without throwing errors when encountering `null` or `undefined`.

```javascript
// Before ES10
const city = user && user.address && user.address.city;

// After ES10
const city = user?.address?.city; // Returns undefined if any part is null/undefined
```

---

### **Nullish Coalescing Operator (??)**

ğŸ“– See: [08. Operators - Nullish Coalescing](../08.%20Operators/1.%20Operators.md#-nullish-coalescing-)

**Syntax:** `value ?? defaultValue`

Returns the right-hand operand when the left-hand is `null` or `undefined` (unlike `||` which checks for any falsy value).

```javascript
// Before ES10
const name = user.name || "Guest"; // "" would become "Guest"

// After ES10
const name = user.name ?? "Guest"; // "" stays "", only null/undefined become "Guest"
```

### **Array.prototype.flat and Array.prototype.flatMap**

ğŸ“– See: [04. Data Types - Arrays](../04.%20Data%20Types/)

Flattens nested arrays.

```javascript
[1, 2, [3, 4]].flat(); // [1, 2, 3, 4]
[1, 2, [3, [4]]].flat(2); // [1, 2, 3, 4]
```

### **Object.fromEntries**

ğŸ“– See: [04. Data Types - Objects](../04.%20Data%20Types/12.%20Objects.md)

Converts array of key-value pairs to object (inverse of `Object.entries`).

```javascript
Object.fromEntries([
  ["a", 1],
  ["b", 2],
]); // { a: 1, b: 2 }
```

### **Promise.allSettled**

ğŸ“– See: [21. Handling Async Js - Promises](../21.%20Handling%20Async%20Js%20-%20Belongs%20to%20ECMAScript/03.%20Promises.md)

Waits for all promises to settle (fulfilled or rejected) and returns results for all.

```javascript
Promise.allSettled([promise1, promise2]).then((results) => {
  // Results include both fulfilled and rejected promises
});
```
