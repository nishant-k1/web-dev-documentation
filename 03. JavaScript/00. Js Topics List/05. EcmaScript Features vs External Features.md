# ECMAScript Language Features (Built-in) vs External Features (Not Part of ECMAScript)

## ECMAScript Language Features (Built-in)

### Fundamentals

- **Variables**: `var`, `let`, `const`
- **Data Types**: `String`, `Number`, `Boolean`, `Object`, `Null`, `Undefined`, `Symbol`, `BigInt`
- **Operators**: Arithmetic, Comparison, Logical, Assignment, etc.
- **Modern Operators**: Optional chaining (`?.`), Nullish coalescing (`??`), Logical assignment (`??=`, `||=`, `&&=`)
- **Control Flow**: `if`, `else`, `switch`, `for`, `while`, `do-while`, `try-catch`
- **Type Coercion and Conversion**: Implicit and explicit type conversion, truthy/falsy values
- **Strict Mode**: `"use strict"` and its effects on code behavior

### Execution Context and Scope

- **Execution Context**: Global context, function context, creation and execution phases (defined in ECMAScript spec)
- **Lexical Environment vs Variable Environment**: How variables are stored and accessed (defined in ECMAScript spec)
- **Scope Chain**: How JavaScript resolves variable references through nested scopes (defined in ECMAScript spec)
- **Scope**: Global scope, function scope, block scope
- **Hoisting**: Variable and function hoisting behavior (defined by ECMAScript spec)
- **Temporal Dead Zone (TDZ)**: Behavior with `let` and `const`

### Functions

- **Functions**: Function declarations, expressions, arrow functions, function scope
- **Arrow Functions vs Regular Functions**: Differences in `this` binding, `arguments` object, hoisting
- **Function Methods**: `call()`, `apply()`, `bind()` - method borrowing and `this` binding
- **Default Parameters**: Function parameters with default values
- **Higher-order Functions**: Functions that accept other functions as arguments or return them
- **Closures**: Lexical scoping, closures, and their role in maintaining state

### Objects and Arrays

- **Objects**: Object literals, constructors, `this` keyword, prototypes
- **Object Static Methods**: `Object.keys()`, `Object.values()`, `Object.entries()`, `Object.assign()`, `Object.freeze()`, `Object.seal()`, `Object.hasOwn()`, `Object.create()`
- **Arrays**: Array literals, methods like `map()`, `filter()`, `reduce()`, `forEach()`, `find()`, `some()`, `every()`, etc.

### Built-in Objects

- **String Methods**: `includes()`, `startsWith()`, `endsWith()`, `slice()`, `substring()`, `split()`, `replace()`, etc.
- **Math Object**: `Math.round()`, `Math.floor()`, `Math.ceil()`, `Math.random()`, `Math.max()`, `Math.min()`, etc.
- **Date Object**: Date creation, manipulation, and formatting methods
- **RegExp**: Regular expressions for pattern matching
- **JSON**: `JSON.parse()`, `JSON.stringify()`

### ES6+ Features

- **Template Literals**: String interpolation, tagged templates
- **Destructuring**: Object and array destructuring, nested destructuring, default values
- **Spread/Rest Operators**: `...` operator for arrays, objects, and function parameters

### Advanced Concepts

- **Prototype-based Inheritance**: Prototype chain and `Object.create()`
- **Classes**: ES6 class syntax, inheritance, static methods, private fields
- **Iterators and Generators**: `for...of`, `Symbol.iterator`, and `function*`, `yield`
- **Map and Set**: `Map`, `Set`, `WeakMap`, `WeakSet` - when to use each
- **Symbol**: Creating unique identifiers, well-known symbols (`Symbol.iterator`, `Symbol.toPrimitive`)
- **BigInt**: Working with large integers beyond `Number.MAX_SAFE_INTEGER`

### Meta-programming

- **Proxy**: Intercepting and customizing object operations
- **Reflection**: `Reflect` API for meta-programming

### Asynchronous Programming

- **Asynchronous Operations Handling Tools**: `Promises`, `async/await`, `Promise.all()`, `Promise.race()`, `Promise.allSettled()`

### Error Handling

- **Error Handling**: `try-catch`, `throw`, `finally`, custom error classes

### Modules and Internationalization

- **Modules**: `import`, `export`, `export default`, dynamic imports
- **Intl API**: Internationalization for dates, numbers, and strings

ECMAScript provides the tools to handle async behavior: Promise, async/await, etc.

But it does not provide the sources of async events like timers or network requests — that's the job of the host environment (browser, Node.js, etc.).

## JavaScript Runtime Environment (JRE) - Not Part of ECMAScript

The JavaScript Runtime Environment includes both the engine mechanisms and the APIs provided by the host environment (browser, Node.js, etc.). These are not part of the ECMAScript language specification:

### Runtime Mechanisms (Implementation Details)

These are implementation details of how JavaScript engines execute ECMAScript code:

- **Call Stack**: How function calls are tracked and executed (implementation detail)
- **Memory Management**: Heap, Stack, Garbage Collector, memory leaks prevention (implementation detail)
- **Event Loop and Task Queues**: Microtask Queue, Macrotask Queue, how asynchronous code is processed (part of host environment, not ECMAScript spec)
- **JavaScript Engine Internals**: JIT Compiler, optimization techniques (implementation-specific, not standardized)

### Runtime Environment APIs (Host Environment APIs)

These are APIs provided by the host environment (browser, Node.js, etc.), not by the ECMAScript specification:

#### DOM and Browser APIs

- **Web APIs/DOM APIs**: DOM manipulation, `document`, `window`, `querySelector()`, `querySelectorAll()`, etc.
- **Event Handling**: `addEventListener()`, `removeEventListener()`, event delegation, event capturing/bubbling
- **Custom Events**: Creating and dispatching custom events with `CustomEvent()`

#### Network and Communication

- **Network APIs**: `fetch()`, `XMLHttpRequest`, AJAX, handling CORS
- **WebSockets**: Real-time bidirectional communication via `WebSocket` API

#### Timers and Animation

- **Timer APIs**: `setTimeout()`, `setInterval()`, `requestAnimationFrame()`, `clearTimeout()`, `clearInterval()`

#### Storage

- **Web Storage APIs**: `localStorage`, `sessionStorage`, storage events
- **IndexedDB**: Client-side database for large amounts of structured data

#### Background Processing

- **Web Workers**: Multi-threading for JavaScript using background workers
- **Service Workers**: Background scripts for offline capabilities, caching, and push notifications

#### Performance and Optimization

- **Performance API**: `performance.now()`, `performance.mark()`, measuring code performance
- **Intersection Observer API**: Efficiently observing element visibility changes
- **Performance Optimization Techniques**: Debouncing, throttling (programming patterns using JavaScript with runtime APIs)

#### Developer Tools

- **Console Methods**: `console.log()`, `console.error()`, `console.table()`, `console.time()`, debugging tools

#### Browser Features

- **Geolocation API**: `navigator.geolocation` for location services
- **Clipboard API**: Reading from and writing to the clipboard
- **Notification API**: Browser notifications

#### Node.js APIs

- **Node.js APIs**: `fs`, `http`, `https`, `path`, `crypto`, `stream`, `events`, `util`, etc.
- **Browser/Node.js Environment**: File I/O, networking, system interactions, etc.

**Note1:** JavaScript engines are implementations of ECMAScript, not part of the language specification
**Note2:** Execution Context, Lexical Environment, and Variable Environment are defined in the ECMAScript specification, but the runtime mechanisms above (Call Stack, Memory Management, Event Loop) are implementation details that vary by engine and host environment.

**Note**: While ECMAScript provides the tools to handle async behavior (Promises, async/await), it does not provide the sources of async events like timers or network requests — that's the job of the host environment (browser, Node.js, etc.). These external APIs are what you typically `await` on when using async/await.
