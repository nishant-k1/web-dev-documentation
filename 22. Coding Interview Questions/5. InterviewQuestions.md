# 🚀 React & JavaScript Interview Questions – Detailed Answers (Senior Developer Level)

Recently I revised some core frontend concepts that are frequently asked in interviews. Sharing detailed answers one by one:
1️⃣ Write a React component and fetch URL with success & error handling
Use useEffect + useState and handle loading, success, and error properly.

```jsx
function Users() {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  useEffect(() => {
    const fetchUsers = async () => {
      try {
        setLoading(true);
        const res = await fetch("https://lnkd.in/eJj6ZuHz");
        if (!res.ok) throw new Error("Failed to fetch");
        const result = await res.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };
    fetchUsers();
  }, []);
  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;
  return data.map((user) => <div key={user.id}>{user.name}</div>);
}
```

2️⃣ 3 siblings want to access same data
Lift state up to the common parent OR use:
React Context (for global/shared state)
Redux/Zustand (for large apps)

3️⃣ Child to Parent Data Handling
Pass a callback from parent to child.

```jsx
function Parent() {
const handleData = (value) => console.log(value);
return <Child sendData={handleData} />;
}

function Child({ sendData }) {
return <button onClick​={() => sendData("Hello")}>Send</button>;
}
```

4️⃣ Handling 5000 Records
Pagination (server-side preferred)
Infinite scroll
Virtualization
Memoization

5️⃣ What is Virtualization?
Rendering only visible items instead of all items in DOM.
Example: react-window
Benefits:
✔ Faster rendering
✔ Reduced memory usage
✔ Better scrolling performance
6️⃣ Memory Leaks in React
Common causes:
Uncleaned event listeners
Uncancelled API calls
Unremoved intervals/timeouts
Solution:
Use cleanup function in useEffect.

7️⃣ Async Operations in JavaScript

async/await (cleanest way)
