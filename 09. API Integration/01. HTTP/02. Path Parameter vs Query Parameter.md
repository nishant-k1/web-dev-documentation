# Path Parameter ve Query Parameter

✅ Standard REST Pattern

1️⃣ Path parameter (most common)
`fetch(`/api/users/${id}`);`

Example:
`fetch("/api/users/42");`

This is the most RESTful and clean way.

It represents:

“Get resource with id = 42”

2️⃣ Query parameter (also valid)
fetch(`/api/users?id=${id}`);

Example:
`fetch("/api/users?id=42");`

Used when:

- Filtering
- Searching
- Optional parameters

But if you're fetching a single resource by primary key, path param is preferred.

🧠 Why GET with Path Param Is Best

Because:

It’s cacheable.
It’s idempotent.
It follows REST conventions.
It works well with CDNs.
It’s predictable.

If asked:

How would you fetch by id in React?

You can say:

“I would use a GET request with the id in the path parameter, such as /api/users/:id, since it aligns with REST principles and keeps the request cacheable and idempotent.”
